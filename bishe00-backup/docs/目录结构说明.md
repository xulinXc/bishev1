## 六、向老师讲解的重点

### 1. AI自动扫描 (main.go:3021-3473)

**核心亮点**: AI驱动的自动化安全扫描

```
用户输入目标 → AI自主决策 → 调用工具执行 → 分析结果 → 循环决策 → 生成报告
```

AI可调用的工具:
- `port_scan` - 端口扫描
- `web_probe` - Web技术栈识别
- `dir_scan` - 目录扫描
- `poc_scan` - 漏洞验证

### 2. JADX MCP集成 (internal/mcp/jadx.go)

**核心亮点**: 通过MCP协议连接JADX，实现APK深度分析

关键技术:
- HTTP/JSON-RPC协议
- Session管理
- SSE流式响应
- 自动重连机制

### 3. AI生成稳定EXP (ai_exp_verify.go)

**核心亮点**: AI生成的EXP代码经过自动验证和迭代修正

工作流程:
1. AI根据漏洞信息生成Python EXP
2. 自动执行验证EXP可用性
3. 如果验证失败，AI分析原因并修正
4. 最多重试5次，直到EXP可用

## 七、为什么这样组织目录

1. **Go项目标准结构**: 使用 `internal/` 存放内部包，符合Go最佳实践
2. **功能内聚**: 相关功能的代码放在一起
3. **便于维护**: 清晰的功能划分便于定位和修改代码
4. **便于讲解**: 目录结构本身就说明了项目的组织方式
