# 端口扫描技术实现详解

## 一、端口扫描原理概述

### 1.1 什么是端口扫描？

**端口扫描（Port Scanning）** 是一种网络探测技术，通过主动连接目标主机的端口来判断：
- 端口是否开放（Open）
- 运行的服务类型（如HTTP、SSH、MySQL）
- 服务的版本信息（通过Banner抓取）

### 1.2 端口状态分类

| 状态 | 说明 | 扫描器反应 |
|------|------|-----------|
| **开放（Open）** | 端口正在监听，接受连接 | 连接成功 |
| **关闭（Closed）** | 端口未监听，拒绝连接 | 收到RST包 |
| **过滤（Filtered）** | 防火墙/IDS阻止探测 | 无响应或ICMP错误 |

### 1.3 扫描技术分类

```
端口扫描技术
├── TCP扫描
│   ├── TCP全连接扫描（Three-way Handshake）
│   ├── TCP半连接扫描（SYN Scan）
│   └── TCP隐蔽扫描（FIN/NULL/Xmas Scan）
└── UDP扫描
    └── UDP探测扫描（发送UDP数据包）
```

## 二、TCP扫描实现

### 2.1 TCP协议基础

TCP建立连接需要**三次握手（Three-way Handshake）**：

```
客户端                    服务器
  |                         |
  |------- SYN ------------>| (1) 客户端发送SYN
  |                         |
  |<---- SYN-ACK -----------| (2) 服务器响应SYN-ACK
  |                         |
  |------- ACK ------------>| (3) 客户端发送ACK
  |                         |
  |===== 连接建立 ==========|
```

**关键点**：
- 如果服务器响应 `SYN-ACK`，说明端口**开放**
- 如果服务器响应 `RST`，说明端口**关闭**
- 如果无响应，说明端口被**过滤**

### 2.2 TCP全连接扫描实现

本项目使用的是**TCP全连接扫描**，即完成完整的三次握手。

#### 核心代码

```go
// TCP扫描核心函数
func scanTCPPort(host string, port int, timeout time.Duration) (bool, string) {
    addr := fmt.Sprintf("%s:%d", host, port)
    
    // 1. 尝试建立TCP连接（完成三次握手）
    conn, err := net.DialTimeout("tcp", addr, timeout)
    
    if err != nil {
        // 连接失败，端口关闭或被过滤
        return false, ""
    }
    
    // 2. 连接成功，端口开放
    defer conn.Close()
    return true, ""
}
```

#### 详细流程

```go
// portScanHandler 的TCP扫描部分
if stype == "tcp" {
    // 1. 建立TCP连接（三次握手）
    conn, err := net.DialTimeout("tcp", addr, timeout)
    
    if err == nil {
        // 2. 连接成功，端口开放
        open = true
        
        // 3. 设置读取超时
        _ = conn.SetDeadline(time.Now().Add(timeout))
        
        // 4. 如果需要抓取Banner
        if req.GrabBanner {
            buf := make([]byte, 256)
            n, _ := conn.Read(buf)
            if n > 0 {
                banner = string(buf[:n])
            }
        }
        
        // 5. 关闭连接（发送FIN）
        _ = conn.Close()
    }
}
```

#### 时序图

```
客户端（扫描器）           目标服务器（80端口）
    |                           |
    |------ SYN -------------->| 第1次握手
    |                           |
    |<--- SYN-ACK -------------| 第2次握手（端口开放）
    |                           |
    |------ ACK -------------->| 第3次握手
    |                           |
    |==== 连接建立 =============|
    |                           |
    |<---- Banner -------------| 服务器主动发送Banner
    |                           |
    |------ FIN -------------->| 客户端关闭连接
    |                           |
    |<----- FIN-ACK -----------|
    |                           |
    |------ ACK -------------->|
    |                           |
    |==== 连接关闭 =============|
```

### 2.3 TCP扫描的优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 实现简单，使用标准库即可 | ❌ 容易被防火墙检测（完整连接） |
| ✅ 准确率高，建立完整连接 | ❌ 速度较慢（需完成三次握手） |
| ✅ 支持Banner抓取 | ❌ 会在目标系统留下日志 |
| ✅ 兼容性好，适用所有平台 | ❌ 需要更多系统资源 |

## 三、UDP扫描实现

### 3.1 UDP协议基础

UDP是**无连接协议**，不需要握手，特点：
- 没有连接状态（不像TCP有ESTABLISHED状态）
- 不保证数据送达（无重传机制）
- 服务器可能不响应，难以判断端口状态

### 3.2 UDP扫描原理

```
客户端                    服务器
  |                         |
  |---- UDP Packet -------->| (1) 发送UDP数据包
  |                         |
  |                         | 情况A：端口开放且服务响应
  |<--- UDP Response -------| (2) 收到响应 → 确认开放
  |                         |
  |                         | 情况B：端口关闭
  |<--- ICMP Port Unreach --| (3) 收到ICMP错误 → 确认关闭
  |                         |
  |                         | 情况C：端口开放但服务不响应
  |       (无响应)          | (4) 超时 → 状态未知
```

**关键点**：
- 收到UDP响应 → 端口**开放**
- 收到ICMP Port Unreachable → 端口**关闭**
- 超时无响应 → 端口**可能开放**或**被过滤**（不确定）

### 3.3 UDP扫描实现

#### 核心代码

```go
// UDP扫描核心函数
func scanUDPPort(host string, port int, timeout time.Duration) (bool, string) {
    addr := fmt.Sprintf("%s:%d", host, port)
    
    // 1. 解析UDP地址
    udpAddr, err := net.ResolveUDPAddr("udp", addr)
    if err != nil {
        return false, ""
    }
    
    // 2. 创建UDP连接
    conn, err := net.DialUDP("udp", nil, udpAddr)
    if err != nil {
        return false, ""
    }
    defer conn.Close()
    
    // 3. 设置超时
    _ = conn.SetDeadline(time.Now().Add(timeout))
    
    // 4. 发送探测数据（简单换行符）
    _, _ = conn.Write([]byte("\n"))
    
    // 5. 尝试读取响应
    buf := make([]byte, 256)
    n, _, err := conn.ReadFrom(buf)
    
    if err == nil && n > 0 {
        // 收到响应，端口开放
        return true, string(buf[:n])
    }
    
    // 无响应或错误，状态未知
    return false, ""
}
```

#### 详细流程

```go
// portScanHandler 的UDP扫描部分
if stype == "udp" {
    // 1. 解析UDP地址
    udpAddr, err := net.ResolveUDPAddr("udp", addr)
    
    if err == nil {
        // 2. 建立UDP"连接"（实际只是绑定地址）
        c, err := net.DialUDP("udp", nil, udpAddr)
        
        if err == nil {
            // 3. 设置读写超时
            _ = c.SetDeadline(time.Now().Add(timeout))
            
            // 4. 发送探测数据（换行符，某些服务可能响应）
            _, _ = c.Write([]byte("\n"))
            
            // 5. 尝试读取响应
            buf := make([]byte, 256)
            n, _, rerr := c.ReadFrom(buf)
            
            if rerr == nil && n > 0 {
                // 收到响应，确认端口开放
                open = true
                banner = string(buf[:n])
            }
            // 注意：没收到响应不代表端口关闭！
            
            // 6. 关闭连接
            _ = c.Close()
        }
    }
}
```

### 3.4 UDP扫描的挑战

| 问题 | 说明 | 解决方案 |
|------|------|----------|
| **假阴性高** | 很多UDP服务不响应探测包 | 发送特定协议的探测数据 |
| **速度慢** | 需要等待超时才能判断 | 增加超时时间（1-2秒） |
| **ICMP限速** | 路由器限制ICMP响应速率 | 降低扫描速度 |
| **准确性低** | 无响应可能是开放或过滤 | 标记为"open\|filtered" |

### 3.5 常见UDP服务的特定探测

对于常见UDP服务，可以发送特定的协议数据：

```go
// DNS (53端口)
dnsQuery := []byte{

    0x01, 0x00, // Flags (standard query)
    0x00, 0x01, // Questions: 1

}

// SNMP (161端口)

    0x30, 0x26, // SEQUENCE
```

## 四、TCP与UDP扫描的关键区别

### 4.1 技术对比

| 对比项 | TCP扫描 | UDP扫描 |
|--------|---------|---------|
| **连接方式** | 面向连接（三次握手） | 无连接（直接发送） |
| **可靠性** | 高（有ACK确认） | 低（无确认机制） |
| **判断依据** | 连接成功/失败 | 响应/ICMP错误/超时 |
| **准确性** | 高（95%+） | 中等（60-70%） |
| **速度** | 中等（300ms超时） | 慢（1-2秒超时） |
| **实现复杂度** | 简单（标准库支持） | 中等（需处理多种情况） |
| **常见端口** | 80, 443, 22, 3306, 6379 | 53, 161, 123, 500 |

### 4.2 代码结构对比

```go
// TCP扫描（简洁）
conn, err := net.DialTimeout("tcp", addr, timeout)
if err == nil {
    // 端口开放
    conn.Close()
}

// UDP扫描（复杂）
udpAddr, _ := net.ResolveUDPAddr("udp", addr)
conn, _ := net.DialUDP("udp", nil, udpAddr)
conn.SetDeadline(time.Now().Add(timeout))
conn.Write([]byte("\n"))
buf := make([]byte, 256)
n, _, err := conn.ReadFrom(buf)
if err == nil && n > 0 {
    // 端口可能开放（不确定）
}
```

### 4.3 适用场景

| 场景 | 推荐方式 | 原因 |
|------|----------|------|
| **Web服务扫描** | TCP | HTTP/HTTPS都是TCP协议 |
| **数据库扫描** | TCP | MySQL、PostgreSQL、Redis都用TCP |
| **DNS服务检测** | UDP | DNS主要使用UDP（53端口） |
| **全端口扫描** | TCP | UDP扫描时间太长 |
| **隐蔽扫描** | TCP SYN（半连接）| 不完成三次握手 |

## 五、Banner抓取技术

### 5.1 什么是Banner？

**Banner（旗标）** 是服务器在连接建立后主动发送的欢迎信息，包含：
- 服务类型（如"SSH-2.0-OpenSSH_8.2p1"）
- 版本号（如"Apache/2.4.41"）
- 操作系统信息（有时包含）

### 5.2 Banner的作用

| 作用 | 说明 | 示例 |
|------|------|------|
| **服务识别** | 识别端口运行的服务 | 80端口返回"HTTP/1.1" |
| **版本检测** | 获取软件版本号 | "nginx/1.18.0" |
| **漏洞扫描** | 根据版本匹配已知漏洞 | "OpenSSH_7.4" → CVE-2018-15473 |
| **指纹识别** | 识别操作系统和设备类型 | "Ubuntu" / "Windows Server" |

### 5.4 Banner抓取实现

#### 核心代码

```go
// Banner抓取函数
func grabBanner(conn net.Conn, timeout time.Duration) string {
    // 1. 设置读取超时
    _ = conn.SetDeadline(time.Now().Add(timeout))
    
    // 2. 创建缓冲区（256字节足够大多数Banner）
    buf := make([]byte, 256)
    
    // 3. 读取服务器主动发送的数据
    n, err := conn.Read(buf)
    
    if err != nil {
        // 读取失败（可能服务不主动发送Banner）
        return ""
    }
    
    if n > 0 {
        // 返回Banner内容（去除不可打印字符）
        return sanitize(string(buf[:n]))
    }
    
    return ""
}

// 清理Banner（去除控制字符）
func sanitize(s string) string {
    var result []rune
    for _, r := range s {
        if r >= 32 && r <= 126 || r == '\n' || r == '\r' {
            result = append(result, r)
        }
    }
    return string(result)
}
```

#### 项目中的实现


### 5.6 Banner抓取的注意事项

| 注意事项 | 说明 | 解决方案 |
|----------|------|----------|
| **超时时间** | Banner可能延迟发送 | 设置1-2秒超时 |
| **缓冲区大小** | Banner长度不固定 | 使用256-1024字节 |
| **二进制数据** | 某些服务返回二进制 | 过滤不可打印字符 |
| **多行Banner** | 某些服务分多次发送 | 循环读取直到超时 |
| **编码问题** | 可能包含特殊字符 | UTF-8解码 + 转义 |

## 六、完整扫描流程示例

### 6.1 TCP扫描+Banner抓取完整流程

```go
// 完整的TCP端口扫描函数
func scanTCPWithBanner(host string, port int, timeout time.Duration) ScanResult {
    addr := fmt.Sprintf("%s:%d", host, port)
    result := ScanResult{Port: port, Proto: "tcp"}
    
    // 第1步：建立TCP连接
    conn, err := net.DialTimeout("tcp", addr, timeout)
    if err != nil {
        // 连接失败
        result.Status = "closed"
        return result
    }
    defer conn.Close()
    
    // 第2步：连接成功，端口开放
    result.Status = "open"
    
    // 第3步：设置读取超时
    _ = conn.SetDeadline(time.Now().Add(timeout))
    
    // 第4步：尝试读取Banner（被动）
    buf := make([]byte, 256)
    n, err := conn.Read(buf)
    
    if err == nil && n > 0 {
        result.Banner = string(buf[:n])
        result.Service = identifyService(port, result.Banner)
        return result
    }
    
    // 第5步：如果没有Banner，主动探测
    result.Banner = activeProbe(conn, port)
    result.Service = identifyService(port, result.Banner)
    
    return result
}

// 根据端口和Banner识别服务
func identifyService(port int, banner string) string {
    // 先根据Banner识别
    if strings.Contains(banner, "HTTP/") {
        return "http"
    } else if strings.Contains(banner, "SSH-") {
        return "ssh"
    } else if strings.Contains(banner, "220") && strings.Contains(banner, "FTP") {
        return "ftp"
    }
    
    // 再根据端口号识别
    switch port {
    case 80, 8080, 8000:
        return "http"
    case 443, 8443:
        return "https"
    case 22:
        return "ssh"
    case 21:
        return "ftp"
    case 3306:
        return "mysql"
    case 6379:
        return "redis"
    default:
        return "unknown"
    }
}
```

### 6.3 并发控制实现

```go
// 信号量模式控制并发数
func portScanConcurrent(host string, ports []int, concurrency int) {
    sem := make(chan struct{}, concurrency) // 信号量，容量=并发数
    var wg sync.WaitGroup
    
    for _, port := range ports {
        wg.Add(1)
        sem <- struct{}{}  // 获取信号量（如果已满会阻塞）
        
        go func(p int) {
            defer func() {
                <-sem      // 释放信号量
                wg.Done()
            }()
            
            // 执行扫描
            result := scanTCPWithBanner(host, p, 300*time.Millisecond)
            
            // 发送结果
            sendResult(result)
        }(port)
    }
    
    wg.Wait() // 等待所有goroutine完成
}
```

**并发控制说明**：
- `concurrency=500`：最多同时扫描500个端口
- 使用信号量（semaphore）限制并发数
- 避免创建过多goroutine导致资源耗尽
- 平衡速度与系统负载

## 七、性能优化技巧

### 7.1 超时设置优化

| 场景 | 推荐超时 | 原因 |
|------|----------|------|
| **局域网扫描** | 100-300ms | 网络延迟低 |
| **公网扫描** | 1-2秒 | 网络延迟高 |
| **UDP扫描** | 2-3秒 | 需等待ICMP响应 |
| **Banner抓取** | 1-2秒 | 服务可能延迟发送 |

### 7.3 常用端口优先扫描

```go
// 常用端口列表（优先扫描）
var commonPorts = []int{
    80, 443, 22, 21, 25, 110, 143, 3306, 3389, 
    1433, 5432, 6379, 27017, 8080, 8000, 8888,
}

// 先扫描常用端口，再扫描全端口
func smartScan(host string, portRange string) {
    // 第1轮：扫描常用端口（快速反馈）
    for _, port := range commonPorts {
        scanPort(host, port)
    }
    
    // 第2轮：扫描完整范围
    allPorts := parsePorts(portRange)
    for _, port := range allPorts {
        if !inCommonPorts(port) {
            scanPort(host, port)
        }
    }
}
```

### 7.4 结果去重与缓存

```go
// 避免重复扫描同一端口
type ScanCache struct {
    m     sync.Mutex
    cache map[string]ScanResult // key: "host:port"
}

func (c *ScanCache) Get(host string, port int) (ScanResult, bool) {
    c.m.Lock()
    defer c.m.Unlock()
    key := fmt.Sprintf("%s:%d", host, port)
    result, exists := c.cache[key]
    return result, exists
}
```

## 十、总结

### 10.1 核心要点

| 要点 | TCP扫描 | UDP扫描 |
|------|---------|---------|
| **连接方式** | `net.DialTimeout("tcp", ...)` | `net.DialUDP(...)` |
| **判断依据** | 连接成功/失败 | 收到响应/ICMP/超时 |
| **超时设置** | 300ms（局域网）/ 1-2s（公网） | 2-3秒 |
| **准确性** | 高（95%+） | 中（60-70%） |
| **Banner抓取** | `conn.Read(buf)` | 部分服务支持 |
| **适用场景** | Web、数据库、SSH等 | DNS、SNMP、NTP等 |

### 10.2 关键代码片段

```go
// TCP扫描
conn, err := net.DialTimeout("tcp", "host:port", timeout)
if err == nil {
    // 端口开放
    buf := make([]byte, 256)
    n, _ := conn.Read(buf)  // 抓取Banner
    banner := string(buf[:n])
}

// UDP扫描
udpAddr, _ := net.ResolveUDPAddr("udp", "host:port")
conn, _ := net.DialUDP("udp", nil, udpAddr)
conn.Write([]byte("\n"))
n, _, err := conn.ReadFrom(buf)
if err == nil && n > 0 {
    // 端口可能开放
}
```

### 10.3 最佳实践

1. **并发控制**：使用信号量限制goroutine数量
2. **超时设置**：根据网络环境动态调整
3. **错误处理**：优雅处理连接失败、超时等异常
4. **资源释放**：使用`defer conn.Close()`确保连接关闭
5. **进度反馈**：通过SSE实时推送扫描进度
6. **Banner解析**：根据Banner识别服务类型和版本
