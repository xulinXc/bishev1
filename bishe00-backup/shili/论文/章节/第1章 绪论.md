# 第1章 绪论

## 1.1 课题研究的目的意义

随着计算机硬件及软件的不断发展，互联网技术已经得到全世界人们的认可，在我们日常生活中的地位也是非常重要的，并渐渐的融入了我们的生活，是我们生活中不可缺少的一部分[1]。企业业务快速向 Web 应用与移动程序迁移，网络安全威胁呈现多样化、复杂化趋势，公开端口暴露、薄弱鉴权接口、第三方组件漏洞与配置缺陷等安全风险集中涌现[2]。在攻防演练、安全评估与渗透测试场景中，安全人员需要在有限时间内完成“资产摸排—脆弱性验证—证据收集”的完整闭环，同时满足等保2.0、数据与隐私合规对测试边界、影响控制与可审计性的严格要求[3]。

然而，现有安全工具生态存在明显的碎片化问题。端口扫描工具（masscan/nmap）与目录枚举工具（ffuf/dirsearch）在各自领域高效，但与业务语义衔接有限，难以形成统一的验证与报告流程[4]；漏洞验证工具 Nuclei 与 X-Ray 规则体系成熟、漏洞覆盖全面，但在多步状态机编排与统一任务管理上成本较高，规则格式互不兼容导致维护负担重[5]；JS 解包工具（packer-fuzzer）、URL 抽取工具（urlfinder）、Android 反编译工具（JADX）、二进制分析工具（IDA Pro）等单点工具丰富，但输入输出分散、缺少统一任务与数据流，人工搬运易丢信息、难沉淀可复用知识[6]；AI 驱动的安全分析能力逐步成熟，但缺乏与扫描工具链的深度集成，难以实现“发现→分析→验证→报告”的自动化闭环[7]。

为了拓展安全评估能力和降低安全评估的成本，一体化安全扫描平台作为一种集成性极强的安全检测系统，可以整合分散的安全工具，实现数据流闭环与合规可审计，为广大安全人员提供一个完整的安全评估平台[8]。平台正慢慢成为安全评估工作的重要工具，或者说成为资产发现与漏洞验证的重要渠道[9]。每个安全人员都可以在上面进行端口扫描、目录枚举、POC 验证、EXP 执行等操作，并且成为了解目标系统安全状况、发现安全漏洞的重要手段[10]。

本系统的功能主要是实现客户端和服务器端的动态交互。NeonScan 系统最基本的功能是可以进行端口扫描，支持 TCP/UDP 协议，自定义端口范围、并发与超时，可选 Banner 抓取[11]；其次，用户可以进行目录扫描，多字典并发与超时控制，判定有效状态码 200/301/302/403，记录响应长度与 Location[12]；为了确保漏洞验证的准确性，用户需要上传 POC/EXP 文件后才能进行漏洞验证，通过SSE事件流实时查看扫描进度和结果[13]。

设计中力求界面友好、简洁，易于操作，并且代码部分一定要避免逻辑错误，程序算法设计需要简单合理，并且程序源码要有较好的可读性，从而利于其他的设计者的阅读[14]。文件操作时也需要保证文件的安全与稳定性，尽量避免操作异常，并要保证程序运行查询的速度与精准性，尽我们最大的能力设计编写出最优程序[15]。平台采用统一的“任务+SSE 事件流”模型实现长任务实时反馈与进度追踪，支持中途停止与 Janitor 自动回收机制，确保系统稳定运行[16]。

## 1.2 国内外研究现状

20 世纪 90 年代，互联网技术逐渐发展起来，网络安全扫描工具的相关应用的主要目标转为网络层探测，逐渐形成现在大家普遍接受的端口扫描概念。国外直接对安全扫描平台进行研究的文献相对较多，大部分的研究表现在漏洞检测以及安全评估等方面。1997 年，Nmap 工具发布，成为最早的端口扫描工具之一。但是，当时互联网并未开始普及，导致当时用户的访问量较少。2000 年，Nessus 漏洞扫描器发布，也是最早的漏洞扫描工具之一。从此之后，随着大家渐渐熟悉网络安全扫描和漏洞检测，安全扫描工具开始飞速发展起来，2005 年，安全扫描工具发展到顶峰。但是由于工具碎片化和发展紊乱，并且受到技术限制的冲击，大量的安全扫描工具功能单一、难以集成。国外的安全扫描工具随之进入调整时期和成熟时期之后，成为现实安全评估的重要工具。

国内的安全扫描工具研究起步相对较晚。2000 年左右，国内开始出现基于 Nmap 的端口扫描工具改进版本。2005 年，国内安全厂商开始开发漏洞扫描平台，但多为商业产品，开源生态相对薄弱。2010 年，随着 Web 应用安全威胁的加剧，国内研究者开始关注 Web 应用安全扫描工具的开发。2015 年，国内开源安全工具开始兴起，但在工具集成与工作流编排方面仍有提升空间。2020 年，随着 AI 技术的发展，国内研究者开始探索 AI 驱动的安全分析能力，但在特定领域如恶意代码检测、漏洞挖掘等方面取得了一定成果。目前，国内安全扫描工具在功能完整性、工具集成度、数据流闭环等方面仍存在不足，需要进一步改进和完善。

## 1.3 论文研究内容

本文主要介绍的是基于 Go 语言的一体化安全扫描平台 NeonScan 的开发流程，介绍了系统开发时所使用的开发框架和开发模式，概括的阐述了开发所使用的相关技术，简要的概述了整体开发过程，详细的介绍了每一个模块的具体实现。

主要设计模块内容如下：

1. 端口扫描模块
包括 TCP/UDP 端口扫描、自定义端口范围、并发与超时控制、Banner 抓取等功能。

2. 目录扫描模块
包括多字典并发扫描、超时控制、有效状态码判定（200/301/302/403）、响应长度与 Location 记录等功能。

3. POC 扫描模块
包括 Legacy 格式解析、X-Ray 表达式引擎、Nuclei 子集解析、状态码/响应体/响应头匹配等功能。

4. EXP 执行模块
包括多步骤编排、变量提取与替换、Cookie 贯穿、重试与休眠机制等功能。

5. Web 探测模块
包括标题采集、协议识别、状态码检测、指纹特征识别（Server/X-Powered-By/Cookie/CMS）、favicon SHA1 检测、robots.txt 检测等功能。

6. WAF 绕过模块
包括大小写变形、URL 编码/双编码、路径填充、参数分割、无害头注入等策略生成与验证功能。

7. JS 资产收集模块
包括可解包性判定、JS 解包、ffuf 字典爆破、urlfinder URL 抓取、结果合并与分类等功能。

8. MCP 集成模块
包括 IDA Pro MCP 集成、JADX MCP 集成、MCP 协议封装等功能。

9. AI 分析模块
包括 AI 网关设计、自动化扫描工作流、智能报告分析等功能。

10. 任务管理模块
包括任务创建、进度追踪、中途停止、自动回收、SSE 事件流推送等功能。
