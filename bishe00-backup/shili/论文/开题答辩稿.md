## 一、选题背景与意义

### 1.1 研究背景

随着互联网技术的快速发展，Web应用和移动应用的安全问题日益突出。传统的安全检测工具通常功能单一、分散，需要安全人员使用多种工具进行组合使用，导致效率低下、学习成本高。同时，现有工具大多存在以下问题：

- **功能分散**：端口扫描、目录爆破、漏洞检测需要使用不同工具
- **实时性差**：缺乏实时进度反馈，用户体验不佳
- **智能化不足**：缺少AI辅助分析，依赖人工判断
- **集成困难**：无法与专业分析工具（IDA Pro、JADX）深度联动

### 1.2 研究意义

本项目旨在开发一个**集成化、智能化、可视化**的安全扫描平台，具有以下意义：

- **提高工作效率**：将多种扫描功能整合到一个平台，减少工具切换
- **增强用户体验**：通过SSE实时推送进度，提供直观的Web界面
- **智能辅助分析**：集成AI能力，帮助安全人员快速定位问题
- **工具深度集成**：通过MCP协议联动IDA Pro、JADX等专业工具

## 四、核心技术原理与实现

### 4.1 SSE实时推送技术

#### **4.1.1 什么是SSE**

SSE（Server-Sent Events，服务器发送事件）是HTML5提供的一种服务器向客户端推送数据的技术。它基于HTTP协议，采用单向通信模式，服务器可以持续向客户端发送数据流。

**核心特点**：
- 基于HTTP长连接，无需WebSocket协议
- 单向推送（服务器→客户端）
- 浏览器原生支持，自动重连
- 文本格式传输（通常是JSON）

#### **4.1.2 为什么选择SSE而不是WebSocket或轮询**

| 技术方案 | 双向通信 | 实现复杂度 | 自动重连 | 应用场景 | 本项目适配度 |
|---------|---------|-----------|---------|---------|------------|
| **轮询** | ✅ | 简单 | ❌ | 低实时性需求 | ❌ 延迟高、资源浪费 |
| **长轮询** | ✅ | 中等 | ❌ | 中等实时性 | ❌ 频繁建连、服务器压力大 |
| **WebSocket** | ✅ | 复杂 | ❌ 需手动 | 聊天、游戏 | ❌ 过度设计，不需要双向 |
| **SSE** | ❌ 单向 | 简单 | ✅ 自动 | 进度推送、通知 | ✅ 完美匹配 |

**本项目选择SSE的原因**：
1. **需求匹配**：扫描任务只需服务器→客户端推送进度，无需客户端频繁发送数据
2. **简单高效**：基于HTTP，无需额外握手，实现简单
3. **自动重连**：浏览器自动处理断线重连，代码量少
4. **资源占用低**：相比WebSocket开销更小

#### **4.1.3 SSE底层原理**

**HTTP响应头设置**：
```http
HTTP/1.1 200 OK
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive
```

**消息格式**：
```
data: {"type":"progress","percent":50,"progress":"500/1000"}\n\n
data: {"type":"find","data":{"port":80,"service":"http"}}\n\n
data: {"type":"end","message":"扫描完成"}\n\n
```

每条消息以`data: `开头，以两个换行符`\n\n`结束。

#### **4.1.4 本项目的SSE实现**

**核心数据结构**：
```go
// 任务对象
type Task struct {
    m       sync.Mutex      // 互斥锁，保护并发访问
    ID      string          // 任务唯一ID（纳秒时间戳）
    Total   int             // 总任务数（如1000个端口）
    Done    int             // 已完成数量
    Created time.Time       // 创建时间
    ch      chan SSEMessage // 容量1024的消息通道
    stop    chan struct{}   // 停止信号通道
    stopped bool            // 停止标志
}

// SSE消息结构
type SSEMessage struct {
    Type     string      `json:"type"`     // start/progress/find/end
    TaskID   string      `json:"taskId"`   // 任务ID
    Progress string      `json:"progress"` // "50/100"
    Percent  int         `json:"percent"`  // 0-100
    Data     interface{} `json:"data"`     // 扫描结果
}
```

**完整实现流程**：

**步骤1：创建任务**
```go
func newTask(total int) *Task {
    id := fmt.Sprintf("t-%d", time.Now().UnixNano())
    t := &Task{
        ID:      id,
        Total:   total,
        Created: time.Now(),
        ch:      make(chan SSEMessage, 1024), // 带缓冲通道
        stop:    make(chan struct{}),
    }
    tasks.Store(id, t)  // 存入全局任务表
    return t
}
```

**步骤2：SSE处理器（核心）**
```go
func sseHandler(w http.ResponseWriter, r *http.Request) {
    // 设置SSE响应头
    w.Header().Set("Content-Type", "text/event-stream")
    w.Header().Set("Cache-Control", "no-cache")
    w.Header().Set("Connection", "keep-alive")
    w.Header().Set("Access-Control-Allow-Origin", "*")
    
    taskID := r.URL.Query().Get("task")
    task, _ := tasks.Load(taskID)
    
    flusher := w.(http.Flusher)
    
    // 持续推送消息
    for {
        select {
        case msg := <-task.ch:  // 从通道接收消息
            data, _ := json.Marshal(msg)
            fmt.Fprintf(w, "data: %s\n\n", data)
            flusher.Flush()  // 立即刷新到客户端
        case <-task.stop:  // 收到停止信号
            return
        case <-r.Context().Done():  // 客户端断开
            return
        }
    }
}
```

**步骤3：工作goroutine推送进度**
```go
func scanPorts(task *Task, ports []int) {
    for i, port := range ports {
        // 执行端口扫描...
        
        // 更新进度
        task.m.Lock()
        task.Done++
        done := task.Done
        task.m.Unlock()
        
        percent := (done * 100) / task.Total
        
        // 发送进度消息
        task.ch <- SSEMessage{
            Type:     "progress",
            Progress: fmt.Sprintf("%d/%d", done, task.Total),
            Percent:  percent,
        }
        
        // 如果发现开放端口，发送结果
        if isOpen {
            task.ch <- SSEMessage{
                Type: "find",
                Data: map[string]interface{}{
                    "port":    port,
                    "service": "http",
                    "banner":  banner,
                },
            }
        }
    }
    
    // 任务完成
    task.ch <- SSEMessage{Type: "end"}
    close(task.stop)
}
```

**并发安全**：

### 4.2 端口扫描技术

#### **4.2.1 端口扫描原理**

端口扫描是通过发送网络数据包到目标主机的指定端口，根据响应判断端口状态的技术。

**端口状态分类**：
- **Open（开放）**：端口正在监听，可以接受连接
- **Closed（关闭）**：端口未监听，但主机可达
- **Filtered（过滤）**：防火墙/路由器丢弃数据包，无法判断状态

#### **4.2.2 TCP端口扫描原理**

**TCP三次握手过程**：
```
客户端                    服务器
  |                         |
  |----SYN(seq=x)---------> |  1. 客户端发送SYN
  |                         |
  | <---SYN+ACK(seq=y)------|  2. 服务器响应SYN+ACK（端口开放）
  |                         |
  |----ACK(seq=x+1)-------> |  3. 客户端发送ACK，完成握手
  |                         |
```

**判断依据**：
- ✅ **三次握手成功** → 端口开放
- ❌ **收到RST（重置）** → 端口关闭
- ❌ **超时无响应** → 端口过滤或主机不可达

**核心实现**：
```go
func scanTCPPort(host string, port int, timeout time.Duration) (bool, string) {
    addr := fmt.Sprintf("%s:%d", host, port)
    
    // 尝试建立TCP连接（完整三次握手）
    conn, err := net.DialTimeout("tcp", addr, timeout)
    if err != nil {
        return false, ""  // 连接失败 → 端口关闭
    }
    defer conn.Close()
    
    // 连接成功 → 端口开放，尝试抓取Banner
    banner := grabBanner(conn)
    return true, banner
}
```

#### **4.2.3 UDP端口扫描原理**

**UDP协议特点**：
- 无连接协议，不需要握手
- 不保证数据送达
- 无状态，无法确认对方是否收到

**扫描原理**：
```
客户端                    服务器
  |                         |
  |----UDP数据包---------->  |  1. 发送UDP探测包
  |                         |
  | <---ICMP Port Unreachable|  2a. 端口关闭 → 返回ICMP错误
  |                         |
  | （无响应）               |  2b. 端口开放 → 可能无响应或返回数据
  |                         |
```

**判断依据**：
- ✅ **收到UDP响应** → 端口开放
- ❌ **收到ICMP端口不可达** → 端口关闭
- ❓ **无响应** → 可能开放，也可能被过滤（不确定）

**核心实现**：
```go
func scanUDPPort(host string, port int, timeout time.Duration) (bool, string) {
    addr := fmt.Sprintf("%s:%d", host, port)
    
    // 解析地址
    udpAddr, _ := net.ResolveUDPAddr("udp", addr)
    
    // 创建UDP连接
    conn, err := net.DialUDP("udp", nil, udpAddr)
    if err != nil {
        return false, ""
    }
    defer conn.Close()
    
    // 设置超时
    conn.SetDeadline(time.Now().Add(timeout))
    
    // 发送探测数据（针对不同服务的特定协议）
    probe := getUDPProbe(port)
    conn.Write([]byte(probe))
    
    // 读取响应
    buf := make([]byte, 1024)
    n, err := conn.Read(buf)
    if err == nil && n > 0 {
        return true, string(buf[:n])  // 有响应 → 端口开放
    }
    
    // 无响应或超时 → 状态不确定
    return false, ""
}

// 根据端口号返回对应的UDP探测包
func getUDPProbe(port int) string {
    switch port {
    case 53:  // DNS
        return "\x00\x00\x10\x00\x00\x00\x00\x00\x00\x00\x00\x00"
    case 161: // SNMP
        return "\x30\x26\x02\x01\x00\x04\x06public\xa0\x19..."
    default:
        return "\x00"  // 空包
    }
}
```

#### **4.2.4 TCP与UDP扫描的关键区别**

| 对比维度 | TCP扫描 | UDP扫描 |
|---------|--------|--------|
| **连接方式** | 三次握手，有连接 | 无连接，直接发包 |
| **可靠性** | 可靠传输，保证送达 | 不可靠，可能丢包 |
| **判断依据** | 握手成功=开放 | 有响应=开放，无响应=不确定 |
| **准确性** | 95%以上 | 60-70%（很多开放端口也不响应） |
| **速度** | 快（300ms超时） | 慢（需2-3秒超时等待） |
| **Banner获取** | 容易（连接建立后直接读） | 困难（需构造特定协议数据） |
| **防火墙检测** | 易被检测（完整握手） | 不易被检测（无状态） |
| **常见服务** | HTTP(80)、SSH(22)、MySQL(3306) | DNS(53)、SNMP(161)、NTP(123) |

**为什么UDP扫描准确性低？**

1. **无响应不等于关闭**：很多UDP服务只在收到特定格式数据时才响应
2. **防火墙影响**：防火墙可能丢弃UDP包，无法区分是防火墙还是端口关闭
3. **ICMP限制**：有些路由器限制ICMP速率，导致"端口不可达"消息丢失

**本项目的解决方案**：
```go
// 对常见UDP端口使用特定探测包
func scanUDPWithProbe(port int) {
    if port == 53 {  // DNS
        sendDNSQuery()
    } else if port == 161 {  // SNMP
        sendSNMPRequest()
    } else {
        sendEmptyPacket()
    }
}
```

#### **4.2.5 Banner抓取技术**

**什么是Banner**：

Banner（服务标识）是服务器在建立连接后发送的标识信息，包含服务类型、版本号等关键信息。

**常见Banner示例**：
```
HTTP/1.1: "Server: nginx/1.18.0"
SSH:      "SSH-2.0-OpenSSH_8.2p1 Ubuntu-4ubuntu0.5"
FTP:      "220 ProFTPD 1.3.5 Server"
MySQL:    "5.7.33-0ubuntu0.18.04.1"
SMTP:     "220 mail.example.com ESMTP Postfix"
```

**Banner的作用**：
1. **服务识别**：判断端口运行的服务类型（HTTP、SSH、FTP等）
2. **版本检测**：获取服务版本号，用于漏洞匹配
3. **指纹识别**：识别操作系统、Web服务器、中间件等

**Banner抓取实现**：

**方式1：被动抓取（服务主动发送）**
```go
func grabBanner(conn net.Conn) string {
    // 设置读取超时
    conn.SetReadDeadline(time.Now().Add(3 * time.Second))
    
    // 读取服务器主动发送的Banner
    buf := make([]byte, 4096)
    n, err := conn.Read(buf)
    if err == nil && n > 0 {
        return string(buf[:n])
    }
    return ""
}
```

**方式2：主动探测（发送HTTP请求）**
```go
func grabHTTPBanner(conn net.Conn) string {
    // 发送HTTP HEAD请求
    conn.Write([]byte("HEAD / HTTP/1.1\r\nHost: target.com\r\n\r\n"))
    
    // 读取响应
    conn.SetReadDeadline(time.Now().Add(3 * time.Second))
    buf := make([]byte, 4096)
    n, _ := conn.Read(buf)
    
    response := string(buf[:n])
    
    // 提取Server头
    if strings.Contains(response, "Server:") {
        lines := strings.Split(response, "\r\n")
        for _, line := range lines {
            if strings.HasPrefix(line, "Server:") {
                return strings.TrimPrefix(line, "Server: ")
            }
        }
    }
    return ""
}
```

**完整扫描流程（TCP + Banner）**：
```go
func fullScan(host string, port int) ScanResult {
    // 1. 尝试连接
    addr := fmt.Sprintf("%s:%d", host, port)
    conn, err := net.DialTimeout("tcp", addr, 3*time.Second)
    if err != nil {
        return ScanResult{Port: port, Open: false}
    }
    defer conn.Close()
    
    // 2. 端口开放，抓取Banner
    banner := grabBanner(conn)
    
    // 3. 如果没有Banner，尝试主动探测
    if banner == "" && (port == 80 || port == 443 || port == 8080) {
        banner = grabHTTPBanner(conn)
    }
    
    // 4. 识别服务类型
    service := identifyService(port, banner)
    
    return ScanResult{
        Port:    port,
        Open:    true,
        Service: service,
        Banner:  banner,
    }
}
```

**注意事项**：
1. **超时设置**：Banner读取需要设置合理超时（2-5秒）
2. **缓冲区大小**：部分服务Banner很长，需足够大的缓冲区
3. **编码问题**：有些Banner非UTF-8编码，需转换
4. **协议适配**：不同服务需要发送不同的探测数据
5. **防火墙干扰**：有些防火墙会修改或隐藏Banner

### 4.3 目录扫描技术

#### **4.3.1 目录扫描原理**

目录扫描（也称目录爆破、路径枚举）是通过字典文件中的路径列表，逐一向Web服务器发送HTTP请求，根据响应状态码判断路径是否存在的技术。

**工作流程**：
```
字典文件 → 读取路径 → 构造URL → HTTP请求 → 检查响应 → 过滤结果 → 实时推送
```

**判断依据**：
- ✅ **200 OK** - 成功访问，路径存在且可访问
- ✅ **403 Forbidden** - 禁止访问，但路径存在（目录存在但无权限）
- ✅ **301/302 Redirect** - 重定向，路径存在
- ❌ **404 Not Found** - 路径不存在
- ❌ **500 Server Error** - 服务器错误（通常过滤）

#### **4.3.3 核心实现**

**读取字典文件**：
```go
func loadDict(dictPath string) ([]string, error) {
    file, err := os.Open(dictPath)
    if err != nil {
        return nil, err
    }
    defer file.Close()
    
    var paths []string
    scanner := bufio.NewScanner(file)
    
    for scanner.Scan() {
        line := strings.TrimSpace(scanner.Text())
        
        // 跳过空行和注释
        if line == "" || strings.HasPrefix(line, "#") {
            continue
        }
        
        paths = append(paths, line)
    }
    
    return paths, nil
}
```

**并发扫描（Worker池模式）**：
```go
func scanDirectory(baseURL string, dictPath string, task *Task) {
    // 1. 读取字典
    paths, _ := loadDict(dictPath)
    task.Total = len(paths)
    
    // 2. 创建HTTP客户端（连接复用）
    client := &http.Client{
        Timeout: 5 * time.Second,
        Transport: &http.Transport{
            MaxIdleConns:        200,
            MaxIdleConnsPerHost: 200,
        },
    }
    
    // 3. 并发控制（200个goroutine）
    sem := make(chan struct{}, 200)
    var wg sync.WaitGroup
    
    // 4. 遍历字典
    for _, path := range paths {
        sem <- struct{}{}  // 获取信号量
        wg.Add(1)
        
        go func(p string) {
            defer func() {
                <-sem  // 释放信号量
                wg.Done()
            }()
            
            // 构造URL
            url := baseURL + "/" + strings.TrimPrefix(p, "/")
            
            // 发送HTTP请求
            resp, err := client.Get(url)
            if err != nil {
                task.IncDone()  // 仅更新进度，不展示
                return
            }
            defer resp.Body.Close()
            
            // 读取响应体长度
            body, _ := io.ReadAll(resp.Body)
            
            // 判断是否展示
            if shouldDisplay(resp.StatusCode, len(body)) {
                task.Send(SSEMessage{
                    Type: "find",
                    Data: map[string]interface{}{
                        "path":   p,
                        "url":    url,
                        "status": resp.StatusCode,
                        "length": len(body),
                        "title":  extractTitle(body),
                    },
                })
            }
            
            // 更新进度
            done, total := task.IncDone()
            task.Send(SSEMessage{
                Type:     "progress",
                Progress: fmt.Sprintf("%d/%d", done, total),
                Percent:  (done * 100) / total,
            })
            
        }(path)
    }
    
    wg.Wait()
    task.Send(SSEMessage{Type: "end"})
}
```

**高级过滤（多维度）**：
```go
type FilterConfig struct {
    StatusCodes    []int   // 包含的状态码
    ExcludeStatus  []int   // 排除的状态码
    MinLength      int     // 最小响应长度
    MaxLength      int     // 最大响应长度
    IncludeKeywords []string // 响应体必须包含的关键词
    ExcludeKeywords []string // 响应体不能包含的关键词
}

func advancedFilter(resp *http.Response, body []byte, config FilterConfig) bool {
    // 1. 状态码过滤
    if len(config.StatusCodes) > 0 {
        found := false
        for _, code := range config.StatusCodes {
            if resp.StatusCode == code {
                found = true
                break
            }
        }
        if !found {
            return false
        }
    }
    
    // 2. 排除状态码
    for _, code := range config.ExcludeStatus {
        if resp.StatusCode == code {
            return false
        }
    }
    
    // 3. 长度过滤
    if config.MinLength > 0 && len(body) < config.MinLength {
        return false
    }
    if config.MaxLength > 0 && len(body) > config.MaxLength {
        return false
    }
    
    // 4. 关键词包含
    bodyStr := string(body)
    for _, keyword := range config.IncludeKeywords {
        if !strings.Contains(bodyStr, keyword) {
            return false
        }
    }
    
    // 5. 关键词排除
    for _, keyword := range config.ExcludeKeywords {
        if strings.Contains(bodyStr, keyword) {
            return false
        }
    }
    
    return true
}
```

## 五、其他核心技术实现

### 5.1 POC漏洞检测技术

#### **5.1.1 什么是POC**

POC（Proof of Concept，概念验证）是用于验证漏洞是否存在的代码或脚本。它通过发送特定的HTTP请求，检查响应是否符合预期特征，从而判断目标是否存在漏洞。

**POC的作用**：
- **漏洞验证**：确认目标是否存在已知漏洞
- **安全评估**：评估系统的安全风险
- **自动化检测**：批量检测大量目标

#### **5.1.2 三种POC格式的区别**

| POC格式 | 语言 | 复杂度 | 表达式引擎 | 兼容性 | 本项目支持 |
|---------|------|-------|-----------|-------|----------|
| **传统POC** | JSON | 简单 | ❌ 无 | 自定义格式 | ✅ 完全支持 |
| **X-Ray POC** | YAML | 中等 | ✅ 复杂表达式 | X-Ray工具 | ✅ 子集支持 |
| **Nuclei POC** | YAML | 复杂 | ✅ Matchers | Nuclei工具 | ✅ 子集支持 |

**传统POC示例**（JSON格式）：
```json
{
    "name": "Struts2-045 RCE",
    "method": "POST",
    "path": "/",
    "headers": {
        "Content-Type": "%{(#_='multipart/form-data')...}"
    },
    "matchBodyAny": ["root:x:0:0", "Windows NT"]
}
```

**X-Ray POC示例**（YAML格式）：
```yaml
name: Struts2-045
rules:
  - method: POST
    path: /
    headers:
      Content-Type: "%{...}"
    expression: |
      response.status == 200 && 
      response.body.bcontains("root:x:0:0")
```

**Nuclei POC示例**（YAML格式）：
```yaml
id: struts2-045
info:
  name: Struts2 RCE
requests:
  - method: POST
    path:
      - "{{BaseURL}}/"
    matchers:
      - type: word
        words:
          - "root:x:0:0"
```

#### **5.1.3 POC执行原理**

**执行流程**：
```
读取POC文件 → 解析格式 → 构造HTTP请求 → 发送请求 → 
检查响应 → 匹配规则 → 判断漏洞 → 推送结果
```

**核心实现**：
```go
// POC结构定义
type POC struct {
    Name         string            `json:"name"`
    Method       string            `json:"method"`      // GET/POST
    Path         string            `json:"path"`        // 请求路径
    Body         string            `json:"body"`        // 请求体
    Headers      map[string]string `json:"headers"`     // 请求头
    MatchBodyAny []string          `json:"matchBodyAny"` // 响应体包含任一
    MatchBodyAll []string          `json:"matchBodyAll"` // 响应体包含所有
    MatchStatus  []int             `json:"matchStatus"`  // 状态码匹配
}

// POC执行函数
func executePOC(baseURL string, poc POC) (bool, string) {
    // 1. 构造URL
    url := baseURL + poc.Path
    
    // 2. 创建HTTP请求
    req, _ := http.NewRequest(poc.Method, url, strings.NewReader(poc.Body))
    
    // 3. 设置请求头
    for k, v := range poc.Headers {
        req.Header.Set(k, v)
    }
    
    // 4. 发送请求
    client := &http.Client{Timeout: 10 * time.Second}
    resp, err := client.Do(req)
    if err != nil {
        return false, "请求失败: " + err.Error()
    }
    defer resp.Body.Close()
    
    // 5. 读取响应体
    body, _ := io.ReadAll(resp.Body)
    bodyStr := string(body)
    
    // 6. 匹配规则
    // 6.1 状态码匹配
    if len(poc.MatchStatus) > 0 {
        matched := false
        for _, status := range poc.MatchStatus {
            if resp.StatusCode == status {
                matched = true
                break
            }
        }
        if !matched {
            return false, "状态码不匹配"
        }
    }
    
    // 6.2 响应体包含任一关键词
    if len(poc.MatchBodyAny) > 0 {
        matched := false
        for _, keyword := range poc.MatchBodyAny {
            if strings.Contains(bodyStr, keyword) {
                matched = true
                break
            }
        }
        if !matched {
            return false, "未匹配到关键词（any）"
        }
    }
    
    // 6.3 响应体包含所有关键词
    if len(poc.MatchBodyAll) > 0 {
        for _, keyword := range poc.MatchBodyAll {
            if !strings.Contains(bodyStr, keyword) {
                return false, "未匹配到关键词（all）: " + keyword
            }
        }
    }
    
    // 7. 所有条件匹配，漏洞存在
    return true, fmt.Sprintf("漏洞存在！状态码:%d", resp.StatusCode)
}
```

#### **5.1.4 表达式引擎实现（X-Ray风格）**

**支持的表达式**：
- `response.status == 200`
- `response.body.bcontains("keyword")`
- `response.headers["Server"].contains("nginx")`
- `response.status == 200 && response.body.bcontains("success")`

**简化实现**：
```go
func evaluateExpression(expr string, resp *http.Response, body []byte) bool {
    // 替换变量
    expr = strings.ReplaceAll(expr, "response.status", fmt.Sprintf("%d", resp.StatusCode))
    
    // 处理 bcontains
    re := regexp.MustCompile(`response\.body\.bcontains\("([^"]+)"\)`)
    matches := re.FindAllStringSubmatch(expr, -1)
    for _, match := range matches {
        keyword := match[1]
        result := strings.Contains(string(body), keyword)
        expr = strings.ReplaceAll(expr, match[0], fmt.Sprintf("%t", result))
    }
    
    // 处理逻辑运算
    // 简化版：直接用字符串替换和eval
    if strings.Contains(expr, "&&") {
        parts := strings.Split(expr, "&&")
        for _, part := range parts {
            if !evaluateSingleCondition(part) {
                return false
            }
        }
        return true
    }
    
    return evaluateSingleCondition(expr)
}
```

### 5.2 WAF绕过技术

#### **5.2.1 什么是WAF**

WAF（Web Application Firewall，Web应用防火墙）是部署在Web应用前端的安全设备，用于检测和阻止恶意请求。

**常见WAF产品**：
- 云WAF：阿里云、腾讯云、Cloudflare
- 硬件WAF：F5、Imperva
- 开源WAF：ModSecurity

#### **5.2.2 WAF绕过原理**

WAF通过规则匹配检测攻击，绕过WAF的核心是对Payload进行**变形**，使其不匹配WAF规则，但仍能被目标应用解析执行。

**六大绕过策略**：

| 策略 | 原理 | 示例 | 适用场景 |
|------|------|------|---------|
| **case** | 大小写变换 | `admin` → `AdMiN` | WAF区分大小写 |
| **urlencode** | URL编码 | `admin` → `%61dmin` | WAF未解码 |
| **doubleencode** | 双重编码 | `admin` → `%2561dmin` | WAF仅解码一次 |
| **pathpad** | 路径填充 | `/admin` → `/./admin` | WAF路径规范化不完整 |
| **headerjunk** | 请求头填充 | 添加`X-Junk: ...` | WAF请求头解析漏洞 |
| **paramsplit** | 参数分割 | `?id=1` → `?id=1&id=1` | WAF取第一个参数 |

#### **5.2.3 核心实现**

**策略应用函数**：
```go
type BypassStrategy string

const (
    Case         BypassStrategy = "case"
    URLEncode    BypassStrategy = "urlencode"
    DoubleEncode BypassStrategy = "doubleencode"
    PathPad      BypassStrategy = "pathpad"
    HeaderJunk   BypassStrategy = "headerjunk"
    ParamSplit   BypassStrategy = "paramsplit"
)

// 应用单个策略
func applyStrategy(payload string, strategy BypassStrategy) string {
    switch strategy {
    case Case:
        return randomCase(payload)
    case URLEncode:
        return url.QueryEscape(payload)
    case DoubleEncode:
        return url.QueryEscape(url.QueryEscape(payload))
    case PathPad:
        return addPathPadding(payload)
    default:
        return payload
    }
}

// 大小写随机变换
func randomCase(s string) string {
    result := []rune{}
    for i, r := range s {
        if i%2 == 0 {
            result = append(result, unicode.ToUpper(r))
        } else {
            result = append(result, unicode.ToLower(r))
        }
    }
    return string(result)
}

// 路径填充
func addPathPadding(path string) string {
    paddings := []string{
        "/./",      // /./ → /
        "//",       // // → /
        "/;/",      // /;/ → /
        "/%2e/",    // /%2e/ → /./
    }
    idx := rand.Intn(len(paddings))
    return strings.ReplaceAll(path, "/", paddings[idx])
}
```

**组合策略**：
```go
func generatePayloadVariants(originalPayload string, strategies []BypassStrategy) []string {
    variants := []string{originalPayload}
    
    // 单策略变体
    for _, strategy := range strategies {
        variants = append(variants, applyStrategy(originalPayload, strategy))
    }
    
    // 双策略组合
    for i := 0; i < len(strategies); i++ {
        for j := i + 1; j < len(strategies); j++ {
            temp := applyStrategy(originalPayload, strategies[i])
            final := applyStrategy(temp, strategies[j])
            variants = append(variants, final)
        }
    }
    
    return variants
}
```

**并发测试**：
```go
func testWAFBypass(baseURL string, originalPayload string, task *Task) {

// 判断是否被WAF拦截
func isBlocked(resp *http.Response, body []byte) bool {
    // 常见WAF拦截特征
    blockKeywords := []string{
        "403 Forbidden",
        "Access Denied",
        "WAF",
        "防火墙",
        "blocked",
    }
    
    bodyStr := string(body)
    for _, keyword := range blockKeywords {
        if strings.Contains(bodyStr, keyword) {
            return true
        }
    }
    
    return resp.StatusCode == 403 || resp.StatusCode == 406
}
```

```go
type MCPClient struct {
    BaseURL    string

}

// 调用MCP工具
func (c *MCPClient) CallTool(toolName string, args map[string]interface{}) (string, error) {
    // 1. 构造JSON-RPC请求
    request := map[string]interface{}{
        "jsonrpc": "2.0",
        "id":      time.Now().UnixNano(),
        "method":  "tools/call",
        "params": map[string]interface{}{
            "name":      toolName,
            "arguments": args,
        },
    }
    
    // 2. 序列化
    reqBody, _ := json.Marshal(request)
    
    // 3. 发送HTTP请求
    resp, err := c.HTTPClient.Post(
        c.BaseURL,
        "application/json",
        bytes.NewReader(reqBody),
    )
    if err != nil {
        return "", err
    }
    defer resp.Body.Close()
    
    // 4. 解析响应
    var result struct {
        JSONRPC string `json:"jsonrpc"`
        ID      int64  `json:"id"`
        Result  struct {
            Content []struct {
                Type string `json:"type"`
                Text string `json:"text"`
            } `json:"content"`
        } `json:"result"`
        Error *struct {
            Code    int    `json:"code"`
            Message string `json:"message"`
        } `json:"error"`
    }
    
    json.NewDecoder(resp.Body).Decode(&result)
    
    // 5. 检查错误
    if result.Error != nil {
        return "", fmt.Errorf("MCP错误: %s", result.Error.Message)
    }
    
    // 6. 提取结果
    if len(result.Result.Content) > 0 {
        return result.Result.Content[0].Text, nil
    }
    
    return "", nil
}
```

#### **5.3.4 MCP与SSE的结合**

```go
func mcpWithSSE(task *Task, mcpClient *MCPClient, toolName string, args map[string]interface{}) {
    // 1. 发送开始消息
    task.Send(SSEMessage{Type: "start", Data: "调用MCP工具: " + toolName})
    
    // 2. 异步调用MCP
    go func() {
        result, err := mcpClient.CallTool(toolName, args)
        if err != nil {
            task.Send(SSEMessage{Type: "error", Data: err.Error()})
            return
        }
        
        // 3. 分块推送大结果
        chunks := splitIntoChunks(result, 1024)
        for i, chunk := range chunks {
            task.Send(SSEMessage{
                Type: "progress",
                Data: chunk,
                Percent: ((i + 1) * 100) / len(chunks),
            })
            time.Sleep(100 * time.Millisecond)  // 控制推送速度
        }
        
        task.Send(SSEMessage{Type: "end"})
    }()
```

#### **5.4.4 AI流式推送技术**

**为什么需要流式推送？**

| 模式 | 响应时间 | 用户体验 | 适用场景 |
|------|---------|---------|---------|
| **普通模式** | 10-30秒 | ❌ 长时间等待，无反馈 | 简单问答 |
| **流式模式** | <1秒开始 | ✅ 实时显示，像打字机 | 长文本生成、代码分析 |

**流式推送原理**：

```
AI模型生成 → 逐Token返回 → SSE推送 → 前端实时显示
  (服务器)      (HTTP流)     (EventSource)   (逐字显示)
```

**OpenAI流式实现**：

```go
func (p *OpenAIProvider) ChatStream(messages []ChatMessage, tools []AITool, controller *StreamController) (string, []AIToolCall, error) {
    // 1. 构建请求（启用流式）
    reqBody := map[string]interface{}{
        "model":    "gpt-4o",
        "messages": convertMessages(messages),
        "stream":   true,  // ✅ 启用流式
    }
    
    // 2. 发送请求
    resp := http.Post("https://api.openai.com/v1/chat/completions", reqBody)
    defer resp.Body.Close()
    
    // 3. 流式解析响应（SSE格式）
    var fullContent strings.Builder
    scanner := bufio.NewScanner(resp.Body)
    
    for scanner.Scan() {
        line := scanner.Text()
        
        // SSE格式：data: {"choices":[{"delta":{"content":"Hello"}}]}
        if !strings.HasPrefix(line, "data: ") {
            continue
        }
        
        dataStr := strings.TrimPrefix(line, "data: ")
        if dataStr == "[DONE]" {
            break
        }
        
        // 解析JSON增量
        var chunk struct {
            Choices []struct {
                Delta struct {
                    Content string `json:"content"`  // 增量内容
                }
            }
        }
        json.Unmarshal([]byte(dataStr), &chunk)
        
        // 提取增量内容
        if len(chunk.Choices) > 0 {
            delta := chunk.Choices[0].Delta.Content
            fullContent.WriteString(delta)
            
            // 立即推送到前端（关键！）
            controller.Send(delta)
        }
    }
    
    return fullContent.String(), nil, nil
}
```

```go
func (p *OllamaProvider) ChatStream(messages []ChatMessage, tools []AITool, controller *StreamController) (string, []AIToolCall, error) {
    // Ollama流式格式：每行一个JSON对象

            Message struct {
                Content string `json:"content"`
            }
            Done bool
        }
        json.Unmarshal([]byte(line), &chunk)
        
        // 提取内容并推送
        if chunk.Message.Content != "" {
            fullContent.WriteString(chunk.Message.Content)
            controller.Send(chunk.Message.Content)
        }
        
        if chunk.Done {
            break
        }
    }
    
    return fullContent.String(), nil, nil
}
```

#### **5.4.5 流式控制器实现**

**StreamController设计**：

```go
type StreamController struct {
    ch      chan string    // 流式内容通道
    abort   chan struct{}  // 中止信号
    aborted bool
    mu      sync.Mutex
}

// 创建控制器
func NewStreamController() *StreamController {
    return &StreamController{
        ch:    make(chan string, 100),  // 缓冲100条消息
        abort: make(chan struct{}),
    }
}

// 发送内容
func (c *StreamController) Send(content string) bool {
    c.mu.Lock()
    defer c.mu.Unlock()
    
    if c.aborted {
        return false
    }
    
    select {
    case c.ch <- content:
        return true
    case <-c.abort:
        return false
    }
}

// 接收内容（SSE推送）
func (c *StreamController) Receive() <-chan string {
    return c.ch
}

// 中止流式输出
func (c *StreamController) Abort() {
    c.mu.Lock()
    defer c.mu.Unlock()
    
    if !c.aborted {
        c.aborted = true
        close(c.abort)
    }
}
```

#### **5.4.6 AI与SSE的结合**

**完整工作流程**：

```
前端发起AI分析请求
  ↓
后端创建Task和StreamController
  ↓
调用AI流式接口（ChatStream）
  ↓
AI返回增量内容 → controller.Send(delta)
  ↓
controller.ch接收 → 转换为SSE事件
  ↓
SSE推送到前端 → EventSource.onmessage
  ↓
前端逐字显示（打字机效果）
```

**后端实现**：

```go
func aiAnalyzeHandler(w http.ResponseWriter, r *http.Request) {
    // 1. 创建Task和StreamController
    task := newTask(1)
    controller := NewStreamController()
    
    // 2. 异步调用AI
    go func() {
        provider := NewOpenAIProvider(apiKey)
        
        messages := []ChatMessage{
            {Role: "user", Content: "分析这个漏洞：SQL注入"},
        }
        
        // 流式聊天
        _, _, err := provider.ChatStream(messages, nil, controller)
        if err != nil {
            task.Send(SSEMessage{Type: "error", Data: err.Error()})
        }
        
        task.Send(SSEMessage{Type: "end"})
    }()
    
    // 3. 转发流式内容到SSE
    go func() {
        for content := range controller.Receive() {
            task.Send(SSEMessage{
                Type: "ai_chunk",
                Data: content,  // 增量内容
            })
        }
    }()
    
    // 4. 返回TaskID
    json.NewEncoder(w).Encode(map[string]string{"taskId": task.ID})
}
```
