# ER图使用说明

## 文件说明

本目录包含两种格式的ER图，适用于不同的使用场景：

### 1. `ER图.puml` - PlantUML格式（UML图版）
- **用途**：生成专业的ER图
- **适用场景**：论文中的ER图、系统设计文档、技术文档
- **使用方法**：
  1. 在线使用：访问 http://www.plantuml.com/plantuml/uml/ 粘贴代码
  2. VS Code插件：安装 PlantUML 插件，直接预览
  3. 命令行工具：使用 PlantUML jar 包生成图片
     ```bash
     java -jar plantuml.jar ER图.puml
     ```
  4. IntelliJ IDEA：安装 PlantUML 插件，直接预览和导出

### 2. `ER图-论文格式.txt` - 文本格式（论文版）
- **用途**：论文中直接使用的文本格式ER图
- **适用场景**：毕业论文、学术论文、技术报告
- **特点**：
  - 使用ASCII字符绘制图形
  - 格式规范，适合直接粘贴到论文中
  - 包含完整的实体定义和关系说明

## 实体说明

### 核心实体

1. **任务（Task）**
   - 系统中的核心实体
   - 表示各种扫描任务（端口扫描、目录扫描、POC扫描等）
   - 包含任务状态、类型、目标等信息

2. **扫描结果（ScanResult）**
   - 存储所有类型的扫描结果
   - 依赖任务实体存在
   - 包含结果类型、数据、时间等信息

3. **POC规则（POC）**
   - 存储各种格式的POC规则
   - 支持Legacy、X-Ray、Nuclei等多种格式
   - 包含规则定义、匹配条件等信息

4. **会话（ChatSession）**
   - AI分析会话实体
   - 可选关联任务（仅AI自动扫描任务）
   - 包含API配置、会话状态等信息

5. **消息（ChatMessage）**
   - 会话消息实体
   - 存储AI对话记录
   - 包含消息角色、内容、时间等信息

6. **上传文件（UploadFile）**
   - 上传的文件实体
   - 支持字典文件、POC文件、二进制文件等
   - 包含文件信息、路径、大小等

7. **指纹规则（FingerprintRule）**
   - Web指纹识别规则实体
   - 用于技术栈识别
   - 包含匹配规则、技术栈名称等

### 关联实体

1. **任务文件关联（TaskFile）**
   - 任务和文件的多对多关系关联表
   - 记录文件的使用方式

2. **POC结果关联（POCResult）**
   - POC规则和扫描结果的多对多关系关联表
   - 记录POC匹配情况

## 关系说明

### 主要关系

1. **任务 → 扫描结果（1对N）**
   - 一个任务可以产生多个扫描结果
   - 强依赖关系，扫描结果必须关联一个任务

2. **任务 → 会话（1对0或1）**
   - 可选关系，只有AI自动扫描任务才关联会话
   - 普通扫描任务不关联会话

3. **会话 → 消息（1对N）**
   - 一个会话包含多条消息
   - 强依赖关系，消息必须关联一个会话

4. **任务 ↔ 上传文件（N对N）**
   - 通过TaskFile关联表实现
   - 一个任务可以使用多个文件，一个文件可以被多个任务使用

5. **POC规则 ↔ 扫描结果（N对N）**
   - 通过POCResult关联表实现
   - 一个POC规则可以匹配多个结果，一个结果可能匹配多个POC规则

### 关系类型

- **依赖关系**：扫描结果依赖任务存在
- **关联关系**：任务与文件、POC与结果的关联
- **组成关系**：会话由消息组成

## 设计特点

### 1. 核心实体设计
- 任务实体是核心，其他实体围绕任务展开
- 扫描结果实体统一存储所有类型的扫描结果

### 2. 灵活的关系设计
- 支持可选关联（任务与会话）
- 使用关联表实现多对多关系
- 支持间接关系（指纹规则通过任务产生结果）

### 3. 扩展性设计
- 实体设计支持未来扩展
- 使用文本字段存储JSON数据，支持灵活的数据结构
- 支持多种文件类型和POC格式

## 在论文中的使用建议

### 方式一：使用UML图（推荐）
1. 使用 PlantUML 工具将 `ER图.puml` 转换为图片
2. 支持的格式：PNG、SVG、PDF等
3. 在论文中插入生成的图片
4. 添加图标题和说明

### 方式二：使用文本格式
1. 打开 `ER图-论文格式.txt`
2. 选择需要的实体定义和关系说明部分
3. 使用等宽字体（如 Courier New）复制粘贴到论文中
4. 调整格式和缩进以适应论文排版

## 注意事项

1. **实体命名**：所有实体使用中文名称，便于理解
2. **主键设计**：所有实体使用字符串类型的主键（TaskID、ResultID等）
3. **外键设计**：明确标注外键关系，保证数据完整性
4. **关系基数**：明确标注关系的基数（1对N、N对N等）
5. **可选关系**：使用"0或1"表示可选关系

## 与包关系图的区别

- **包关系图**：展示代码层面的包/模块关系，关注功能模块的组织
- **ER图**：展示数据层面的实体关系，关注数据的存储和组织

两者从不同角度描述系统：
- 包关系图描述"怎么做"（系统功能）
- ER图描述"存储什么"（数据结构）

## 修改建议

如果需要对ER图进行修改：

1. **添加新实体**：
   - 在PlantUML文件中添加实体定义
   - 定义实体之间的关系
   - 更新关系说明文档

2. **修改实体属性**：
   - 根据实际需求修改属性列表
   - 保持命名规范的一致性
   - 更新属性说明

3. **调整关系**：
   - 根据实际业务逻辑调整关系
   - 更新关系基数标注
   - 更新关系说明文档
