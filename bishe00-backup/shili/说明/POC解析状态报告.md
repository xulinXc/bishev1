### 解析能力

✅ **程序已经可以正常解析和使用 80.8% 的内置 POC**

#### 已改进的解析功能
1. ✅ **标准 X-Ray 格式支持** - 完全支持标准的 X-Ray POC 格式
2. ✅ **Nuclei 格式支持** - 支持大部分 Nuclei 格式的 POC
3. ✅ **缩进错误修复** - 能够处理 rules 在 info 下的缩进错误
4. ✅ **Expression 提取** - 能够从文件内容中提取 expression 字段
5. ✅ **Nuclei 变体支持** - 支持 matchers 在 request 内部的格式

#### 解析失败的主要原因
1. **YAML 语法错误** (约 100+ 个文件)
   - 如 `haproxy-status-info-leak.yaml` 有两个 `rules:` 定义
   - 这些是文件格式错误，需要手动修复

2. **非标准格式** (约 200+ 个文件)
   - 某些 nuclei_pocs 目录下的文件使用了不标准的格式
   - 使用了 Nuclei 的高级特性（如 `detections` 替代 `matchers`）
   - 需要更复杂的解析逻辑

3. **格式不完整** (约 50+ 个文件)
   - 缺少必要的字段（如缺少 `expression` 或 `rules`）
   - 文件可能不完整或已损坏

1. **无法解析的 POC 会被自动跳过**
   - 程序不会因为个别 POC 解析失败而停止
   - 只会加载可解析的 POC 进行扫描

2. **建议定期更新**
   - 修复格式错误的 POC 文件
   - 更新 POC 库到最新版本

3. **自定义 POC**
   - 如果内置 POC 无法满足需求，可以使用自定义 POC 目录
   - 确保自定义 POC 符合标准格式
