# JADX MCP 端口说明

## 端口区分

在 JADX MCP 系统中，有两个不同的端口，它们有不同的用途：

### 1. 端口 8650 - JADX AI MCP Plugin（插件端口）

- **用途**：JADX AI MCP Plugin 在 JADX 中运行时的端口
- **作用**：MCP 服务器（jadx-mcp-server）与 JADX 插件之间的通信端口
- **访问者**：只有 MCP 服务器会连接这个端口，用于获取 JADX 中的代码、类等信息
- **默认值**：8650（可通过 `--jadx-port` 参数修改）

### 2. 端口 8651/9999 - FastMCP HTTP 服务器（MCP 服务器端口）

- **用途**：FastMCP 服务器对外提供的 HTTP 接口端口
- **作用**：Web 应用或其他客户端连接 MCP 服务器的端口
- **访问者**：您的 Web 应用（通过 Go 后端）连接这个端口
- **默认值**：8651（FastMCP 默认），但可以通过 `--port` 参数自定义

## 为什么需要两个端口？

```
┌─────────────┐         ┌──────────────────┐         ┌──────────────┐
│   Web应用    │ ──────► │  MCP服务器        │ ──────► │ JADX Plugin  │
│ (Go后端)    │  8651   │ (jadx-mcp-server) │  8650   │ (端口 8650)   │
└─────────────┘         └──────────────────┘         └──────────────┘
```

1. **Web 应用 → MCP 服务器（端口 8651/9999）**
   - 您的 Go 后端通过这个端口连接 MCP 服务器
   - 发送 MCP 协议请求（tools/list, tools/call 等）

2. **MCP 服务器 → JADX Plugin（端口 8650）**
   - MCP 服务器通过这个端口连接 JADX 中的插件
   - 获取 APK 分析结果、类代码等信息

## 配置说明

### 如果使用默认端口启动 MCP 服务器：

```powershell
cd "G:\jadx-gui-1.5.3-with-jre-win\jadx-mcp-server-v3.3.5\jadx-mcp-server"
python jadx_mcp_server.py --http
# 默认使用端口 8651
```

**Web 界面配置**：`http://127.0.0.1:8651`

### 如果自定义端口启动（如 9999）：

```powershell
python jadx_mcp_server.py --http --port 9999
```

**Web 界面配置**：`http://127.0.0.1:9999`

### JADX Plugin 端口（通常不需要修改）：

```powershell
python jadx_mcp_server.py --http --port 9999 --jadx-port 8650
```

**注意**：`--jadx-port` 参数指定的是 JADX Plugin 的端口（默认 8650），这个不需要在 Web 界面中配置，只有 MCP 服务器需要知道。

## 总结

- ✅ **Web 界面需要配置的端口**：FastMCP HTTP 服务器端口（8651 或自定义的如 9999）
- ❌ **Web 界面不需要配置的端口**：JADX Plugin 端口（8650），这是 MCP 服务器内部使用的

**当前您的配置**：
- MCP 服务器运行在：`http://127.0.0.1:9999`（您手动启动时指定的）
- JADX Plugin 运行在：`http://127.0.0.1:8650`（JADX 中显示的，由 MCP 服务器连接）

**建议**：
如果您想让配置更简单，可以让 MCP 服务器使用默认端口 8651 启动，这样 Web 界面就默认使用 `http://127.0.0.1:8651`。
