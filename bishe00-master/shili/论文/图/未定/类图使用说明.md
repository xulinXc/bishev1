# 类图使用说明

## 文件说明

- **类图.puml**: PlantUML格式的类图源文件
- **类图.png**: 从PlantUML生成的类图图片（需要手动生成）

## 类图结构说明

本类图展示了NeonScan项目的核心类结构，包括以下几个主要模块：

### 1. 任务管理模块

- **Task**: 任务管理类，用于管理异步扫描任务的执行状态和进度
  - 属性：ID、Total、Done、Created、通道和停止信号
  - 方法：IncDone()、Send()、Close()

- **SSEMessage**: Server-Sent Events消息结构体，用于向前端实时推送扫描进度和结果

### 2. 扫描请求模块

包含各种扫描功能的请求结构体：

- **PortScanReq**: 端口扫描请求
- **DirScanReq**: 目录扫描请求
- **PocScanReq**: POC扫描请求
- **WAFBypassReq**: WAF绕过测试请求
- **ExpExecReq**: EXP验证请求
- **WebProbeReq**: Web探测请求
- **AIAnalyzeReq**: AI分析请求

### 3. POC规则模块

支持多种POC格式：

- **POC**: 传统POC格式
- **XRPOC**: X-Ray风格的POC格式
  - 包含 **XRInfo**（POC信息）、**XRRule**（规则）、**XRRequest**（请求）
- **NucleiPOC**: Nuclei格式的POC
  - 包含 **NucleiInfo**（信息）、**NucleiRequest**（请求）、**NucleiMatcher**（匹配器）、**NucleiHeaders**（请求头）

### 4. EXP验证模块

- **ExpSpec**: EXP规范，定义完整的EXP
- **ExpStep**: EXP执行步骤，包含请求参数、验证规则和变量提取规则
- **Validation**: 响应验证规则，定义如何判断HTTP响应是否符合预期
- **ExtractRule**: 变量提取规则，用于从HTTP响应中提取变量
- **StatusList**: 状态码列表，支持字符串和整数两种格式

### 5. WAF绕过模块

- **PayloadVariant**: Payload变体及其使用的策略

### 6. MCP接口模块

- **MCPConnection**: MCP连接接口（通用接口）
  - **IDAMCPConnection**: IDA Pro MCP连接实现
  - **JADXMCPConnection**: JADX MCP连接实现
- **ChatSession**: 聊天会话接口（通用接口）
  - **BaseChatSession**: 基础聊天会话实现
    - **IDAChatSession**: IDA聊天会话
    - **JADXChatSession**: JADX聊天会话
- **MCPTool**: MCP工具定义

### 7. AI Provider模块

- **AIProvider**: AI提供商接口
  - **OpenAIProvider**: OpenAI API实现
  - **DeepSeekProvider**: DeepSeek API实现
  - **AnthropicProvider**: Anthropic (Claude) API实现
  - **OllamaProvider**: Ollama本地AI API实现

- **ChatStreamProvider**: 流式聊天接口
  - 由 **OpenAIProvider**、**DeepSeekProvider**、**OllamaProvider** 实现

- **ChatMessage**: 聊天消息结构体
- **AITool**: AI工具定义（用于函数调用）
- **AIToolCall**: AI工具调用
- **StreamController**: 流控制器，用于控制流式输出的中止和消息传递

### 8. 其他模块

- **FingerprintRule**: 指纹识别规则
- **FingerprintData**: 指纹数据集合
- **DictInfo**: 字典信息

## 类之间的关系

### 组合关系

- `ExpSpec` 包含多个 `ExpStep`
- `ExpStep` 包含 `Validation` 和多个 `ExtractRule`
- `XRPOC` 包含 `XRInfo` 和多个 `XRRule`
- `NucleiPOC` 包含 `NucleiInfo`、多个 `NucleiRequest` 和多个 `NucleiMatcher`

### 继承关系

- `BaseChatSession` 实现 `ChatSession` 接口
- `IDAChatSession` 和 `JADXChatSession` 继承 `BaseChatSession`
- `IDAMCPConnection` 和 `JADXMCPConnection` 实现 `MCPConnection` 接口
- `OpenAIProvider`、`DeepSeekProvider`、`AnthropicProvider`、`OllamaProvider` 实现 `AIProvider` 接口

### 使用关系

- 所有扫描请求都使用 `Task` 类来管理任务进度
- `BaseChatSession` 使用 `AIProvider` 和 `MCPConnection`
- `BaseChatSession` 包含多个 `ChatMessage`
- `ChatMessage` 可以包含多个 `AIToolCall`

## 生成类图图片

### 方法1：使用PlantUML在线工具

1. 访问 [PlantUML在线服务器](http://www.plantuml.com/plantuml/uml/)
2. 复制 `类图.puml` 文件内容
3. 粘贴到在线编辑器
4. 生成并下载PNG图片

### 方法2：使用本地PlantUML

```bash
# 安装PlantUML（需要Java）
# Windows: 下载 plantuml.jar
# Linux/Mac: sudo apt-get install plantuml 或 brew install plantuml

# 生成PNG图片
java -jar plantuml.jar 类图.puml

# 或使用命令行
plantuml 类图.puml
```

### 方法3：使用VS Code插件

1. 安装VS Code插件 "PlantUML"
2. 打开 `类图.puml` 文件
3. 按 `Alt+D` 预览类图
4. 右键选择 "Export Current Diagram" 导出为PNG

> 图X-X展示了NeonScan系统的核心类结构。系统采用模块化设计，主要包括任务管理、扫描请求、POC规则、EXP验证、WAF绕过、MCP接口和AI Provider等模块。任务管理模块采用Task类统一管理所有异步扫描任务，通过SSE（Server-Sent Events）机制实时向前端推送进度和结果。扫描请求模块定义了各种扫描功能的请求结构体，包括端口扫描、目录扫描、POC扫描等。POC规则模块支持多种POC格式，包括传统POC、X-Ray风格POC和Nuclei格式POC。MCP接口模块提供了与IDA Pro和JADX的集成能力，通过MCPConnection接口实现统一的外部工具连接。AI Provider模块支持多种AI服务提供商，包括OpenAI、DeepSeek、Anthropic和Ollama，实现了统一的AI分析接口。

## 注意事项

1. **类图复杂度**：当前类图展示了所有主要类，如果觉得过于复杂，可以：
   - 只展示核心类和接口
   - 按照功能模块分别绘制多个类图
   - 简化类的属性，只显示关键方法

2. **关系标注**：类图中的关系使用标准UML符号：
   - 实线箭头：继承/实现关系
   - 实线菱形：组合关系
   - 虚线箭头：使用/依赖关系

3. **扩展性**：如果需要添加新的类或关系，可以直接编辑 `类图.puml` 文件
