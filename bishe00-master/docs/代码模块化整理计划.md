# 代码模块化整理计划

## 当前问题

根目录 `.go` 文件过多（8个），且 EXP 相关代码分散在多个文件中。

## 推荐整理方案

### 方案：将 EXP 相关代码整合到 `pkg/exp/` 包

**目标结构**：
```
pkg/exp/
├── types.go         # ExpSpec, ExpStep, Validation, ExtractRule 等核心类型
├── exec.go          # execExp, expExecHandler 等执行逻辑
├── generator.go     # generatePythonFromExpSpec, buildExpKeyInfo
├── ai.go           # newAIProvider, aiGenPythonFromExpHandler
├── ai_verify.go    # GenerateExpWithVerification, verifyExp
├── ai_batch.go     # aiGenPythonFromExpBatchHandler
└── autoscan.go     # generateAndVerifyExpScripts
```

**需要提取的代码**：
1. `main.go` 中的 Task/SSEMessage 类型和相关函数 → 新建 `pkg/task/`
2. `main.go` 中的 `generatePythonScriptsForVulnerabilities` → `pkg/exp/generator.go`
3. `exp.go` → `pkg/exp/types.go` + `pkg/exp/exec.go`
4. `ai_exp.go` → `pkg/exp/ai.go`
5. `ai_exp_verify.go` → `pkg/exp/ai_verify.go`
6. `ai_exp_batch.go` → `pkg/exp/ai_batch.go`
7. `main_autoscan_exploit_verify.go` → `pkg/exp/autoscan.go`

**修改点**：
1. 所有文件的 `package` 声明改为 `package exp`
2. 添加 `bishe/pkg/task` 导入
3. 修改 `main.go` 的路由注册：
```go
import "bishe/pkg/exp"

mux.HandleFunc("/ai/exp/python", exp.AIGenPythonFromExpHandler)
mux.HandleFunc("/ai/exp/python/batch", exp.AIGenPythonFromExpBatchHandler)
mux.HandleFunc("/exp/exec", exp.ExpExecHandler)
```

---

## 执行步骤

### 步骤 1：创建 pkg/task/ 包

```bash
mkdir -p pkg/task
```

提取 `main.go` 中的：
- SSEMessage 结构体
- Task 结构体
- newTask, getTask, stopTask, finishTask
- safeSend, writeSSE

### 步骤 2：创建 pkg/exp/ 包

```bash
mkdir -p pkg/exp
```

按上述结构组织文件。

### 步骤 3：修改 main.go

1. 删除已移动的代码
2. 修改 import 路径
3. 修改路由注册

### 步骤 4：验证编译

```bash
go build .
```

---

## 注意事项

1. **依赖关系复杂**：`exp.go` 的 `expExecHandler` 使用了 `newTask` 等函数，需要先完成 `pkg/task/` 包
2. **类型引用**：所有 EXP 相关文件都使用 `ExpSpec` 等类型，需要保持类型定义一致
3. **测试**：重构后需要全面测试确保功能不变
