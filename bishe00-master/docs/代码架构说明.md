## 五、为什么 main.go 这么大

`main.go` 之所以有 5564 行，是因为：

1. **Go语言不允许循环依赖**
   - scan需要依赖task中的SSEMessage
   - ai需要依赖scan的执行结果
   - 所有包都依赖mcp的AI接口
   - 所以不能按理想的方式完全拆分

2. **内聚性考虑**
   - HTTP handler和其依赖的逻辑放在同一个包
   - 方便维护和理解业务流程

3. **代码是按照功能模块清晰分段的**
   - 每个模块有明确的注释标题
   - 模块之间有足够的空行分隔
   - 便于向老师解释每个部分的功能
