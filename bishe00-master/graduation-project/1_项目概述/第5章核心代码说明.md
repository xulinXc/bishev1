## 5.1 任务管理模块核心代码

### 核心代码片段
- `newTask` 函数：创建新任务
- `sseHandler` 函数：SSE事件流处理器

### 为什么是核心代码
任务管理模块是整个系统的**基础设施**，所有扫描模块都依赖它来管理任务生命周期和实时通信。没有任务管理，系统无法实现异步扫描和实时进度推送。

### 核心在哪里
1. **`newTask`函数**：
   - **唯一性保证**：使用`time.Now().UnixNano()`生成唯一任务ID，避免任务冲突
   - **并发安全**：使用`mu.Lock()`保护全局任务表，防止并发写入导致的数据竞争
   - **资源初始化**：创建带缓冲区的通道（容量1024），平衡内存占用和性能
   - **停止机制**：初始化`stop`通道，支持优雅停止任务

2. **`sseHandler`函数**：
   - **实时通信**：通过Server-Sent Events实现服务器主动推送，无需前端轮询
   - **连接管理**：监听`r.Context().Done()`检测客户端断开，及时释放资源
   - **任务同步**：监听`t.stop`通道，任务停止时发送结束消息
   - **消息转发**：从任务通道`t.ch`读取消息并转发给前端，实现解耦

### 作用
- **任务生命周期管理**：统一管理所有扫描任务的创建、执行、停止和清理
- **实时进度推送**：通过SSE实现前端实时显示扫描进度和结果
- **资源隔离**：每个任务有独立的通道和停止信号，互不干扰
- **内存管理**：通过任务清理守护进程（Janitor）自动清理过期任务，防止内存泄漏

### 完整代码及逐行说明

#### 1. newTask 函数

```go
func newTask(total int) *Task {
    // 第67行：生成唯一任务ID（基于当前时间戳）
    // time.Now().UnixNano() 获取纳秒级时间戳，确保ID唯一性
    // fmt.Sprintf 格式化为字符串，前缀"t-"便于识别任务ID
    id := fmt.Sprintf("t-%d", time.Now().UnixNano())
    
    // 第69-76行：创建任务结构体并初始化所有字段
    t := &Task{
        ID:      id,                                    // 任务唯一标识符
        Total:   total,                                 // 任务总数（如需要扫描的端口数）
        Done:    0,                                     // 已完成数量，初始为0
        Created: time.Now(),                           // 任务创建时间，用于后续清理判断
        ch:      make(chan SSEMessage, 1024),          // SSE消息通道，容量1024，带缓冲区避免阻塞
        stop:    make(chan struct{}),                  // 停止信号通道，用于优雅停止任务
    }
    
    // 第78-80行：将任务添加到全局任务表（并发安全）
    mu.Lock()        // 获取全局互斥锁，防止并发写入
    tasks[id] = t    // 将任务添加到全局任务映射表
    mu.Unlock()      // 释放锁
    
    return t         // 返回创建的任务对象
}
```

**关键点说明**：
- **第68行**：使用纳秒级时间戳确保ID唯一性，避免任务冲突
- **第74行**：通道容量1024，平衡内存占用和性能，避免消息丢失
- **第75行**：`stop`通道用于发送停止信号，所有goroutine监听此通道
- **第78-80行**：使用互斥锁保护全局任务表，确保并发安全

```go
func sseHandler(w http.ResponseWriter, r *http.Request) {
    // 第234行：从URL查询参数中获取任务ID
    // 前端通过 /events?task=t-xxx 连接SSE流
    id := r.URL.Query().Get("task")
    
    // 第235-238行：验证任务ID是否存在
    if id == "" {
        http.Error(w, "missing task", http.StatusBadRequest)  // 返回400错误
        return
    }

    // 第241-245行：从全局任务表中获取任务对象
    t, ok := getTask(id)  // getTask是线程安全的函数
    if !ok {
        http.Error(w, "task not found", http.StatusNotFound)  // 任务不存在，返回404
        return
    }

    // 第248-250行：设置SSE响应头
    // Content-Type: text/event-stream 是SSE协议要求的响应头
    w.Header().Set("Content-Type", "text/event-stream")  // SSE协议标识
    w.Header().Set("Cache-Control", "no-cache")          // 禁用缓存
    w.Header().Set("Connection", "keep-alive")            // 保持连接

    // 第252行：获取请求上下文，用于检测客户端断开连接
    ctx := r.Context()
    
    // 第253行：创建15秒的心跳定时器，定期发送ping消息保持连接活跃
    ticker := time.NewTicker(15 * time.Second)
    defer ticker.Stop()  // 函数返回时停止定时器，释放资源

    // 第257-259行：发送开始消息，通知前端任务已开始
    if err := writeSSE(w, SSEMessage{
        Type:     "start",                              // 消息类型：开始
        TaskID:   id,                                   // 任务ID
        Progress: fmt.Sprintf("0/%d", t.Total),        // 进度文本：0/总数
        Percent:  0,                                    // 进度百分比：0%
    }); err != nil {
        return  // 发送失败则直接返回
    }

    // 第262-298行：主循环，监听多个通道事件
    for {
        select {
        case <-ctx.Done():
            // 第264-266行：客户端断开连接
            // ctx.Done() 在客户端关闭连接时会被触发
            return  // 直接返回，释放资源

        case <-t.stop:
            // 第267-275行：任务停止信号
            // 当任务被停止时，t.stop通道会被关闭，触发此case
            _ = writeSSE(w, SSEMessage{
                Type:     "end",                        // 消息类型：结束
                TaskID:   id,
                Progress: fmt.Sprintf("%d/%d", func() int { 
                    t.m.Lock()                          // 获取任务锁
                    defer t.m.Unlock()                 // 确保释放锁
                    return t.Done                       // 返回已完成数量
                }(), t.Total),
                Percent:  100,                          // 进度100%
            })
            return  // 发送结束消息后返回

        case <-ticker.C:
            // 第276-280行：心跳消息（每15秒发送一次）
            // 保持连接活跃，防止代理服务器或浏览器超时断开
            if err := writeSSE(w, SSEMessage{
                Type:   "ping",                         // 消息类型：心跳
                TaskID: id,
            }); err != nil {
                return  // 发送失败则返回
            }

        case msg, ok := <-t.ch:
            // 第281-296行：接收任务消息
            // t.ch 是任务的消息通道，扫描过程中会向此通道发送消息
            if !ok {
                // 第283-291行：通道已关闭
                // 通道关闭表示任务已完成，发送结束消息
                _ = writeSSE(w, SSEMessage{
                    Type:     "end",
                    TaskID:   id,
                    Progress: fmt.Sprintf("%d/%d", func() int { 
                        t.m.Lock()
                        defer t.m.Unlock()
                        return t.Done 
                    }(), t.Total),
                    Percent:  100,
                })
                return
            }
            // 第293-296行：转发消息到前端
            // 将任务发送的消息（progress、find等）转发给前端
            if err := writeSSE(w, msg); err != nil {
                return  // 发送失败则返回
            }
        }
    }
}
```

**关键点说明**：
- **第234行**：从URL参数获取任务ID，前端通过此ID连接对应的SSE流
- **第252行**：使用请求上下文检测客户端断开，及时释放资源
- **第253行**：心跳机制防止连接超时，15秒间隔平衡性能和资源
- **第262-298行**：select语句监听多个通道，实现事件驱动的异步处理
- **第281行**：从任务通道读取消息，实现任务和SSE连接的解耦

## 5.2 端口扫描模块核心代码

### 核心代码片段
- TCP/UDP连接建立逻辑
- Banner抓取逻辑
- 并发控制和进度更新

### 为什么是核心代码
端口扫描是**最基础的扫描功能**，其他扫描模块（如Web探测、目录扫描）通常需要先知道开放端口。TCP和UDP两种协议的扫描逻辑完全不同，需要分别处理。

### 核心在哪里
1. **TCP扫描**：
   - **连接探测**：使用`net.DialTimeout`建立TCP连接，通过连接成功与否判断端口开放状态
   - **超时控制**：设置连接超时，避免长时间等待
   - **Banner抓取**：连接成功后立即读取服务欢迎信息，用于服务识别

2. **UDP扫描**：
   - **无连接协议**：UDP是无连接协议，需要先`ResolveUDPAddr`解析地址
   - **数据包发送**：发送空数据包（`\n`）触发服务响应
   - **响应检测**：通过`ReadFrom`读取响应，有响应则端口开放

3. **并发控制**：
   - **信号量模式**：使用`sem`通道限制并发数，防止资源耗尽
   - **WaitGroup同步**：确保所有goroutine完成后再结束任务
   - **进度实时更新**：每次扫描完成后调用`IncDone()`更新进度，通过SSE推送

### 作用
- **端口发现**：快速识别目标主机开放的端口和服务
- **服务识别**：通过Banner信息识别服务类型和版本
- **性能优化**：通过并发控制平衡扫描速度和资源占用
- **实时反馈**：扫描过程中实时推送结果，用户无需等待全部完成

### 完整代码及逐行说明

#### portScanHandler 函数核心部分

```go
func portScanHandler(w http.ResponseWriter, r *http.Request) {
    // 第390-394行：解析HTTP请求体中的JSON数据
    var req PortScanReq
    if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
        http.Error(w, err.Error(), http.StatusBadRequest)  // JSON解析失败，返回400错误
        return
    }
    
    // 第395-399行：解析端口规格字符串（如"1-1024,80,443"）
    ports := parsePorts(req.Ports)  // 解析为端口列表
    if len(ports) == 0 {
        http.Error(w, "no ports", http.StatusBadRequest)  // 没有有效端口，返回400错误
        return
    }
    
    // 第400-409行：设置并发数和超时时间（带默认值）
    cc := req.Concurrency
    if cc <= 0 {
        cc = 500  // 默认并发数500
    }
    timeout := time.Duration(req.TimeoutMs)
    if timeout <= 0 {
        timeout = 300 * time.Millisecond  // 默认超时300毫秒
    } else {
        timeout = time.Duration(req.TimeoutMs) * time.Millisecond  // 转换为time.Duration类型
    }
    
    // 第410行：创建新任务，总数为端口数量
    t := newTask(len(ports))
    
    // 第411行：启动goroutine异步执行扫描（不阻塞HTTP响应）
    go func() {
        // 第412行：创建信号量通道，容量为并发数，用于限制同时扫描的端口数
        sem := make(chan struct{}, cc)
        
        // 第413行：创建WaitGroup，用于等待所有goroutine完成
        var wg sync.WaitGroup
        
        // 第414-417行：确定扫描类型（TCP或UDP），默认TCP
        stype := strings.ToLower(strings.TrimSpace(req.ScanType))
        if stype == "" {
            stype = "tcp"  // 默认TCP扫描
        }
        
        // 第418-483行：遍历所有端口，为每个端口启动一个goroutine
        for _, p := range ports {
            // 第419-423行：检查任务是否已停止（支持中途停止）
            select {
            case <-t.stop:
                break  // 任务已停止，跳出循环
            default:
            }
            
            // 第424行：WaitGroup计数器+1，表示启动了一个goroutine
            wg.Add(1)
            
            // 第425行：获取信号量（如果通道已满则阻塞，限制并发数）
            sem <- struct{}{}
            
            // 第426行：保存端口号到局部变量（避免闭包问题）
            pp := p
            
            // 第427行：启动goroutine扫描单个端口
            go func() {
                defer wg.Done()  // goroutine结束时，WaitGroup计数器-1
                
                // 第429-434行：再次检查任务是否已停止
                select {
                case <-t.stop:
                    <-sem  // 释放信号量
                    return // 直接返回，不执行扫描
                default:
                }
                
                // 第435行：构造目标地址（IP:端口）
                addr := fmt.Sprintf("%s:%d", req.Host, pp)
                
                // 第436-437行：初始化端口状态和Banner变量
                open := false      // 端口是否开放
                var banner string  // Banner信息
                
                // 第438-453行：UDP扫描逻辑
                if stype == "udp" {
                    // 第439行：解析UDP地址
                    udpAddr, err := net.ResolveUDPAddr("udp", addr)
                    if err == nil {
                        // 第441行：创建UDP连接
                        c, err := net.DialUDP("udp", nil, udpAddr)
                        if err == nil {
                            // 第443行：设置读取超时
                            _ = c.SetDeadline(time.Now().Add(timeout))
                            
                            // 第444行：发送空数据包（\n）触发服务响应
                            _, _ = c.Write([]byte("\n"))
                            
                            // 第445-450行：尝试读取响应
                            buf := make([]byte, 256)  // 256字节缓冲区
                            n, _, rerr := c.ReadFrom(buf)  // 读取响应
                            if rerr == nil && n > 0 {
                                open = true                    // 有响应，端口开放
                                banner = string(buf[:n])       // 保存Banner信息
                            }
                            
                            // 第451行：关闭UDP连接
                            _ = c.Close()
                        }
                    }
                } else {
                    // 第454-467行：TCP扫描逻辑
                    // 第455行：建立TCP连接（带超时）
                    conn, err := net.DialTimeout("tcp", addr, timeout)
                    if err == nil {
                        open = true  // 连接成功，端口开放
                        
                        // 第458行：设置读取超时
                        _ = conn.SetDeadline(time.Now().Add(timeout))
                        
                        // 第459-464行：如果启用Banner抓取，尝试读取服务信息
                        if req.GrabBanner {
                            buf := make([]byte, 256)  // 256字节缓冲区
                            n, _ := conn.Read(buf)    // 读取Banner（非阻塞）
                            if n > 0 {
                                banner = string(buf[:n])  // 保存Banner信息
                            }
                        }
                        
                        // 第466行：关闭TCP连接
                        _ = conn.Close()
                    }
                }
                
                // 第469-480行：更新进度并发送结果
                // 第469行：增加已完成计数（线程安全）
                d, tot := t.IncDone()
                
                // 第470行：计算进度百分比
                percent := int(math.Round(float64(d) / float64(tot) * 100))
                
                // 第471行：创建进度消息
                msg := SSEMessage{
                    Type:     "progress",                    // 消息类型：进度
                    TaskID:   t.ID,
                    Progress: fmt.Sprintf("%d/%d", d, tot), // 进度文本：已完成/总数
                    Percent:  percent,                       // 进度百分比
                }
                
                // 第472-479行：如果端口开放，创建find消息
                if open {
                    msg.Type = "find"  // 消息类型：发现
                    data := map[string]interface{}{
                        "port":   pp,        // 端口号
                        "status": "open",    // 状态：开放
                        "proto":  stype,     // 协议：tcp或udp
                    }
                    if banner != "" {
                        data["banner"] = banner  // 如果有Banner，添加到数据中
                    }
                    msg.Data = data
                }
                
                // 第480行：发送消息到SSE通道（线程安全）
                safeSend(t, msg)
                
                // 第481行：释放信号量，允许下一个goroutine执行
                <-sem
            }()
        }
        
        // 第484行：等待所有goroutine完成
        wg.Wait()
        
        // 第485行：标记任务完成
        finishTask(t.ID)
    }()
    
    // 第487-488行：立即返回任务ID给前端（不等待扫描完成）
    w.Header().Set("Content-Type", "application/json")
    json.NewEncoder(w).Encode(map[string]string{"taskId": t.ID})
}
```

**关键点说明**：
- **第412行**：信号量模式限制并发数，防止创建过多goroutine导致资源耗尽
- **第413行**：WaitGroup确保所有扫描完成后才结束任务
- **第425行**：获取信号量后再启动goroutine，精确控制并发数
- **第426行**：保存端口号到局部变量，避免闭包捕获循环变量的问题
- **第455行**：`DialTimeout`带超时参数，避免长时间等待关闭的端口
- **第469行**：`IncDone()`是线程安全的，多个goroutine并发调用不会出错
- **第480行**：`safeSend`内部处理了并发安全问题，可以安全地从多个goroutine调用

## 5.3 目录扫描模块核心代码

### 核心代码片段
- 字典文件读取和解析
- 工作池模式实现
- 状态码判定逻辑

### 为什么是核心代码
目录扫描是**Web安全测试的基础**，用于发现隐藏的目录和文件。工作池模式是Go语言中处理大量并发任务的经典模式，体现了系统的并发设计能力。

### 核心在哪里
1. **字典解析**：
   - **文件读取**：从文件系统读取字典文件，支持多文件合并
   - **路径规范化**：自动为路径添加`/`前缀，确保URL格式正确
   - **注释过滤**：跳过以`#`开头的注释行，提高可读性

2. **工作池模式**：
   - **任务分发**：通过`jobs`通道分发任务，实现生产者-消费者模式
   - **Worker复用**：固定数量的worker goroutine处理所有任务，避免创建过多goroutine
   - **优雅停止**：监听`t.stop`信号，支持中途停止扫描

3. **状态码判定**：
   - **有效状态码**：200（成功）、301/302（重定向）、403（禁止访问）都视为有效发现
   - **信息记录**：记录响应长度和Location头，用于后续验证和报告
   - **错误处理**：网络错误不终止整个任务，继续扫描其他路径

### 作用
- **资产发现**：发现Web应用的隐藏目录和文件
- **漏洞入口**：找到管理后台、配置文件等敏感路径
- **性能优化**：工作池模式平衡并发数和资源占用，避免系统过载
- **结果记录**：记录详细的响应信息，便于后续分析和验证

func dirScanHandler(w http.ResponseWriter, r *http.Request) {
    // 第506-510行：解析HTTP请求体
    var req DirScanReq
    if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
        http.Error(w, err.Error(), http.StatusBadRequest)
        return
    }
    
    // 第512-521行：验证并规范化基础URL
    u := strings.TrimSpace(req.BaseURL)
    if u == "" {
        http.Error(w, "missing baseUrl", http.StatusBadRequest)
        return
    }
    low := strings.ToLower(u)
    // 如果没有协议前缀，自动添加http://
    if !strings.HasPrefix(low, "http://") && !strings.HasPrefix(low, "https://") {
        u = "http://" + u
    }
    req.BaseURL = u
    
    // 第523-540行：定义字典文件读取函数
    var paths []string
    readOne := func(fp string) error {
        // 第525行：读取字典文件内容
        b, err := os.ReadFile(fp)
        if err != nil {
            return err
        }
        // 第529行：按行分割文件内容
        for _, line := range strings.Split(string(b), "\n") {
            // 第530行：去除首尾空白字符
            p := strings.TrimSpace(line)
            // 第531-533行：跳过空行和注释行（以#开头）
            if p == "" || strings.HasPrefix(p, "#") {
                continue
            }
            // 第534-536行：如果路径不以/开头，自动添加
            if !strings.HasPrefix(p, "/") {
                p = "/" + p
            }
            // 第537行：添加到路径列表
            paths = append(paths, p)
        }
        return nil
    }
    
    // 第542-551行：读取内置字典
    if len(req.BuiltinDicts) > 0 {

                // 第601-609行：创建HTTP请求

                        Type:     "progress",
                        TaskID:   t.ID,
                        Progress: fmt.Sprintf("%d/%d", d, tot),

                    }

                }

                d, tot := t.IncDone()
                percent := int(math.Round(float64(d) / float64(tot) * 100))
                msg := SSEMessage{
                    Type:     "progress",
                    TaskID:   t.ID,
                    Progress: fmt.Sprintf("%d/%d", d, tot),
                    Percent:  percent,
                }
                
                // 第626行：判断是否为有效状态码（200/301/302/403）
                if status == 200 || status == 403 || status == 301 || status == 302 {
                    msg.Type = "find"  // 有效发现
                    msg.Data = map[string]interface{}{
                        "path":     path,     // 路径
                        "url":      url,      // 完整URL
                        "status":   status,   // 状态码
                        "location": loc,      // 重定向地址
                        "length":   bodyLen,  // 响应体长度
                    }
                }
                
                // 第630行：发送消息
                safeSend(t, msg)
            }
        }
        
        // 第634-644行：启动worker goroutine（工作池）
        wN := cc
        if wN < 1 {
            wN = 1  // 至少1个worker
        }
        if wN > 1000 {
            wN = 1000  // 最多1000个worker，防止资源耗尽
        }
        wg.Add(wN)  // WaitGroup计数器设置为worker数量
        for i := 0; i < wN; i++ {
            go worker()  // 启动worker goroutine
        }
        
        // 第646-653行：向jobs通道发送任务（生产者）
        for _, p := range paths {
            select {
            case <-t.stop:
                break  // 任务已停止，停止发送任务
            default:
            }
            jobs <- p  // 发送路径到jobs通道
        }
        close(jobs)  // 关闭通道，通知所有worker任务已发送完毕
        
        // 第654行：等待所有worker完成
        wg.Wait()
        
        // 第655行：标记任务完成
        finishTask(t.ID)
    }()
    
    // 第657-658行：立即返回任务ID
    w.Header().Set("Content-Type", "application/json")
    json.NewEncoder(w).Encode(map[string]string{"taskId": t.ID})
}

func evalRuleExpression(expr string, status int, body []byte, headers http.Header) bool {
    // 第1676-1680行：严格性检查
    // 如果响应体为空且表达式包含bcontains，直接返回false
    if len(body) == 0 && strings.Contains(expr, "bcontains") {
        return false  // 空响应无法进行包含匹配
    }

    ex := expr  // 复制表达式，避免修改原表达式
    
    // 第1684-1694行：替换 response.status == N 格式
    // 正则表达式匹配 response.status == 数字 的模式
    reStatus := regexp.MustCompile(`response\.status\s*==\s*(\d+)`)
    ex = reStatus.ReplaceAllStringFunc(ex, func(s string) string {
        m := reStatus.FindStringSubmatch(s)  // 提取状态码数字
        if len(m) == 2 {
            n, _ := strconv.Atoi(m[1])  // 转换为整数
            if status == n {
                return "true"  // 状态码匹配，替换为true
            }
        }
        return "false"  // 状态码不匹配，替换为false
    })
    
    // 第1696-1710行：替换 response.body.bcontains("...") 格式
    // 支持单引号和双引号，非贪婪匹配
    reContains := regexp.MustCompile(`response\.body\.bcontains\(b?['\"]([\s\S]*?)['\"]\)`)
    ex = reContains.ReplaceAllStringFunc(ex, func(s string) string {
        m := reContains.FindStringSubmatch(s)  // 提取要搜索的字符串
        if len(m) == 2 {
            needle := m[1]  // 要搜索的关键词
            if needle == "" {
                return "false"  // 关键词为空，返回false
            }
            // 检查响应体是否包含关键词
            if len(body) > 0 && strings.Contains(string(body), needle) {
                return "true"
            }
        }
        return "false"
    })
    
    // 第1712-1727行：替换 r'pattern'.bmatches(response.body) 格式（正则匹配）
    reMatchesR := regexp.MustCompile(`r['"]([\s\S]*?)['"]\.bmatches\(response\.body\)`)
    ex = reMatchesR.ReplaceAllStringFunc(ex, func(s string) string {
        m := reMatchesR.FindStringSubmatch(s)
        if len(m) == 2 {
            pat := m[1]  // 正则表达式模式
            if pat == "" || len(body) == 0 {
                return "false"
            }
            re, e := regexp.Compile(pat)  // 编译正则表达式
            if e == nil && re.Match(body) {  // 匹配响应体
                return "true"
            }
        }
        return "false"
    })
    
    // 第1773-1775行：规范化布尔值（True/False -> true/false）
    ex = strings.ReplaceAll(ex, "True", "true")
    ex = strings.ReplaceAll(ex, "False", "false")
    
    // 第1775行：调用布尔表达式求值函数
    return evalBoolExpr(ex)
}

func evalBoolExpr(expr string) bool {
    ex := strings.TrimSpace(expr)
    if ex == "" {
        return false  // 空表达式返回false
    }
    
    // 第1802-1815行：递归处理括号（优先级最高）
    for {
        re := regexp.MustCompile(`\([^()]*\)`)  // 匹配最内层括号
        loc := re.FindStringIndex(ex)
        if loc == nil {
            break  // 没有括号了，退出循环
        }
        inner := ex[loc[0]+1 : loc[1]-1]  // 提取括号内的表达式
        val := evalBoolExpr(inner)         // 递归求值
        replacement := "false"
        if val {
            replacement = "true"
        }
        // 用求值结果替换括号表达式
        ex = ex[:loc[0]] + replacement + ex[loc[1]:]
    }
    
    // 第1817-1842行：处理OR和AND运算符
    orTerms := splitByOperator(ex, "||")  // 按||分割
    result := false
    for _, term := range orTerms {
        andTerms := splitByOperator(term, "&&")  // 按&&分割
        andVal := true
        hasValidTerm := false
        for _, f := range andTerms {
            v := strings.TrimSpace(f)
            if v == "" {
                continue
            }
            hasValidTerm = true
            if v != "true" {
                andVal = false  // AND条件：有一个false就是false
                break
            }
        }
        if hasValidTerm && andVal {
            result = true  // OR条件：有一个true就是true
            break
        }
    }
    return result
}

### 作用

- **自动化测试**：自动化执行复杂的攻击流程，提高测试效率

### 完整代码及逐行说明

func execExp(base string, spec ExpSpec, client *http.Client) (bool, int, int) {
    // 第349-352行：初始化变量
    matchedSteps := 0                    // 匹配的步骤数
    lastStatus := 0                      // 最后一步的HTTP状态码
    vars := make(map[string]string)      // 变量存储（用于步骤间传递数据）
    cookieJar := make(map[string]string) // Cookie存储（自动管理Cookie）

    // 第355行：按顺序执行每个步骤
    for _, st := range spec.Steps {
        // 第357-364行：构建目标URL（支持绝对URL和相对路径）
        rawPath := substVars(st.Path, vars)  // 替换路径中的变量占位符
        low := strings.ToLower(strings.TrimSpace(rawPath))
        url := ""
        // 如果路径是完整的URL（http://或https://开头），直接使用
        if strings.HasPrefix(low, "http://") || strings.HasPrefix(low, "https://") {
            url = rawPath
        } else {
            // 否则拼接基础URL和路径
            url = strings.TrimRight(base, "/") + rawPath
        }
        
        // 第365行：替换请求体中的变量占位符
        body := substVars(st.Body, vars)
        
        // 第366-369行：确定HTTP方法（默认GET）
        m := strings.ToUpper(strings.TrimSpace(st.Method))
        if m == "" {
            m = "GET"
        }
        
        // 第370-373行：计算重试次数（retry+1，至少1次）
        attempts := st.Retry + 1
        if attempts <= 0 {
            attempts = 1
        }
        
        // 第374-376行：初始化响应相关变量
        var resp *http.Response
        var err error
        var b []byte
        
        // 第377-414行：重试循环
        for i := 0; i < attempts; i++ {
            // 第379-382行：构建请求体（每次重试都重新构建，避免body reader被消耗）
            var reqBody io.Reader
            if body != "" {
                reqBody = strings.NewReader(body)  // 创建字符串Reader
            }
            
            // 第383-391行：创建HTTP请求
            req, e := http.NewRequest(m, url, reqBody)
            if e != nil {
                err = e
                // 请求创建失败，如果配置了重试延迟，则休眠后继续下一次尝试
                if st.RetryDelayMs > 0 {
                    time.Sleep(time.Duration(st.RetryDelayMs) * time.Millisecond)
                }
                continue  // 继续下一次重试
            }
            
            // 第393-395行：设置请求头（替换变量占位符）
            for k, v := range st.Headers {
                req.Header.Set(k, substVars(v, vars))  // 替换header值中的变量
            }
            
            // 第397-403行：添加Cookie（从cookieJar中获取之前步骤设置的Cookie）
            if len(cookieJar) > 0 {
                var pairs []string
                for ck, cv := range cookieJar {
                    pairs = append(pairs, fmt.Sprintf("%s=%s", ck, cv))  // 格式：key=value
                }
                req.Header.Set("Cookie", strings.Join(pairs, "; "))  // 多个Cookie用;分隔
            }
            
            // 第404行：发送HTTP请求
            resp, err = client.Do(req)
            
            // 第405-410行：如果请求成功，读取响应体并跳出重试循环
            if err == nil {
                b, _ = io.ReadAll(resp.Body)  // 读取响应体
                resp.Body.Close()              // 关闭响应体
                lastStatus = resp.StatusCode   // 保存状态码
                break  // 请求成功，跳出重试循环
            }
            
            // 第411-413行：如果请求失败且配置了重试延迟，休眠后继续重试
            if st.RetryDelayMs > 0 {
                time.Sleep(time.Duration(st.RetryDelayMs) * time.Millisecond)
            }
        }
        
        // 第415-417行：如果所有重试都失败，跳过此步骤
        if err != nil {
            continue  // 继续下一个步骤
        }
        
        // 第418-420行：如果配置了步骤间休眠，执行休眠
        if st.SleepMs > 0 {
            time.Sleep(time.Duration(st.SleepMs) * time.Millisecond)
        }
        
        // 第422-428行：捕获Set-Cookie头，更新cookieJar
        for _, sc := range resp.Header.Values("Set-Cookie") {
            parts := strings.Split(sc, ";")  // Cookie值可能包含;分隔的属性（如expires、path等）
            kv := strings.SplitN(parts[0], "=", 2)  // 只取第一个=之前的部分（key=value）
            if len(kv) == 2 {
                cookieJar[strings.TrimSpace(kv[0])] = strings.TrimSpace(kv[1])  // 保存到cookieJar
            }
        }
        
        // 第430行：从响应中提取变量（BodyRegex、HeaderRegex）
        extractVars(resp, b, st.Extract, vars)
        
        // 第431-433行：验证步骤是否成功
        if validate(resp, b, st.Validate) {
            matchedSteps++  // 验证通过，匹配步骤数+1
        }
    }
    
    // 第436行：成功判定：所有步骤都匹配且至少有一个步骤
    return matchedSteps == len(spec.Steps) && len(spec.Steps) > 0, matchedSteps, lastStatus
}
```

**关键点说明**：
- **第351行**：`vars` map用于在步骤间传递变量，支持复杂的多步骤利用链
- **第352行**：`cookieJar`自动管理Cookie，实现会话保持，支持需要登录的漏洞利用
- **第357行**：`substVars`函数替换变量占位符（如`{{var}}`），实现动态请求构建
- **第377-414行**：重试机制提高成功率，每次重试都重新构建请求，避免body reader被消耗
- **第397-403行**：自动管理Cookie，将之前步骤设置的Cookie自动添加到后续请求中
- **第422-428行**：从Set-Cookie头提取Cookie，更新cookieJar，实现会话保持
- **第430行**：`extractVars`函数通过正则表达式从响应中提取变量，用于后续步骤
- **第431-433行**：`validate`函数验证响应是否符合预期，只有验证通过才计入匹配步骤
- **第436行**：所有步骤都匹配才判定为成功，确保完整的利用链执行成功

前端界面是**用户交互的核心**，SSE连接是实现实时更新的关键机制。没有SSE连接，前端无法实时显示扫描进度和结果。

### 核心在哪里
1. **SSE连接管理**：
   - **EventSource创建**：创建EventSource连接到SSE端点
   - **消息处理**：处理不同类型的SSE消息（start、progress、find、end）
   - **连接清理**：任务结束时自动关闭连接

2. **任务状态管理**：
   - **运行中任务跟踪**：跟踪所有运行中的任务
   - **持久化**：将运行中的任务保存到localStorage，页面刷新后自动恢复
   - **状态同步**：同步任务状态，确保UI与后端一致

3. **事件处理**：
   - **进度更新**：根据progress消息更新进度条
   - **结果显示**：根据find消息显示扫描结果
   - **任务结束**：根据end消息清理任务状态

### 作用

#### startTask 函数

function startTask(key, taskId, onMsg){
    // 第260行：关闭已存在的EventSource连接（避免重复连接）
    // running[key]?.es 使用可选链操作符，如果不存在则不执行
    if(running[key]?.es) {
        try{ 
            running[key].es.close()  // 关闭旧连接
        }catch{}  // 忽略关闭错误（连接可能已关闭）
    }
    
    // 第261-264行：创建新的SSE连接
    // sse函数内部创建EventSource，连接到 /events?task=taskId
    const es = sse(taskId, (m)=>{
        // 消息回调函数
        if(m.type==='end'){
            // 任务结束，清理运行中的任务记录
            delete running[key]
            persistRunningTasks()  // 持久化到localStorage
        }
        onMsg(m)  // 调用外部传入的消息处理函数
    })
    
    // 第265行：保存任务信息到running对象
    running[key] = { 
        id: taskId,  // 任务ID
        es           // EventSource对象
    }
    
    // 第266行：持久化运行中的任务到localStorage
    persistRunningTasks()
}
```

**关键点说明**：
- **第260行**：关闭旧连接避免重复连接，使用可选链操作符安全访问
- **第261行**：`sse`函数封装了EventSource的创建和消息处理逻辑
- **第262-264行**：消息回调函数处理end消息，自动清理任务状态
- **第265行**：`running`对象跟踪所有运行中的任务，key是任务类型（如'ps'、'ds'）
- **第266行**：`persistRunningTasks`将任务状态保存到localStorage，支持页面刷新后恢复
