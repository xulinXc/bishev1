# 毕业论文摘要

随着业务快速向 Web 与移动程序迁移，公开端口暴露、薄弱鉴权接口、第三方组件漏洞等安全风险集中涌现。攻防演练要求在有限时间内完成“资产摸排—脆弱性验证—证据收集”的完整闭环，同时满足等保2.0与数据合规要求。现有工具生态中，端口/目录探测工具与业务语义衔接有限；Nuclei 与 X-Ray 规则体系在多步编排上成本较高；JS 解包、反编译等单点工具输入输出分散，缺少统一数据流，难沉淀可复用知识。

针对上述问题，本文设计并实现了一个轻量一体化安全扫描平台 NeonScan。平台采用 Go 1.21 单进程 HTTP 服务架构，将端口扫描、目录枚举、POC/EXP 验证、Web 指纹探测、WAF 绕过与 JS/API 资产收集纳入统一后端与前端，采用统一的“任务+SSE 事件流”模型实现实时反馈。平台兼容 Legacy、X-Ray 与 Nuclei 子集三种 POC 规则格式，支持状态码、响应体、响应头的布尔组合匹配；EXP 执行模块支持多步骤编排、变量提取与 Cookie 贯穿。通过将 JS/API 清单自动回流至目录扫描、POC 扫描模块，形成“资产发现→验证→报告”的数据流闭环。平台通过 MCP 协议集成 IDA Pro 与 JADX，集成 AI 能力提供自动化扫描工作流，采用目录隔离与定期清理机制满足合规要求。实验结果表明，目录命中率提升 ≥30%，平均总耗时降低 ≥40%，定位瓶颈时间成本降低 ≥50%。平台有效解决了工具碎片化、数据不闭环与合规可审计不足的问题。

**关键词**：网络安全扫描；漏洞验证；任务编排；MCP 协议；AI 安全分析
