## 二、MCP协议到底是什么

MCP（Model Context Protocol）是Anthropic公司提出的一个协议，目的是让AI模型能够调用外部工具。

### 2.1 为什么需要MCP？

举个例子：我想让AI帮我分析一个APK文件。问题是：
- AI只知道文字，不知道APK文件是什么
- APK需要用专门的工具（JADX）来反编译
- AI需要"调用"JADX来获取信息

传统方式下，AI只能靠文字描述来猜测APK的内容。但有了MCP协议，AI可以直接调用JADX这个外部工具，把工具返回的结果整合到自己的回答中。

### 2.2 MCP的工作原理

MCP协议定义了三个核心概念：

**1. 工具（Tools）**
外部工具要事先注册到MCP服务器上，声明自己有什么功能。比如JADX MCP服务器注册了这些工具：
- `decompile_class` - 反编译某个类
- `get_class_list` - 获取所有类列表
- `search_string` - 搜索字符串

**2. 资源（Resources）**
工具可以访问的数据。比如JADX可以访问当前加载的APK文件内容。

**3. 提示（Prompts）**
预设的对话模板。

### 2.3 JSON-RPC通信

MCP协议底层使用的是JSON-RPC，这是一种简单的远程过程调用协议。

举个例子，当我调用JADX的`get_class_list`工具时，发送的请求是这样的：

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "get_class_list",
    "arguments": {}
  }
}
```

服务器返回：

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": [
      {
        "type": "text",
        "text": "com.example.app.MainActivity\ncom.example.app.LoginActivity\n..."
      }
    ]
  }
}
```

**我的工作量**：理解MCP协议的通信格式，自己用Go语言实现了一个MCP客户端，能够：
1. 发送JSON-RPC格式的请求
2. 解析返回的JSON响应
3. 处理连接超时、重试等情况

## 三、JADX MCP是怎么工作的

### 3.1 整体架构

```
┌─────────────────┐
│   我们的系统     │
│  (Go后端)       │
└────────┬────────┘
         │ HTTP + JSON-RPC
         │
┌────────▼────────┐
│  JADX MCP Server │  ← 这是别人写的Python程序
│  (jadx_mcp_server.py)
└────────┬────────┘
         │
┌────────▼────────┐
│    JADX GUI      │  ← JADX编辑器
│  (Java程序)      │
└─────────────────┘
```

JADX MCP Server是一个中间层，它做的事情是：
1. 接收我们系统发送的JSON-RPC请求
2. 调用JADX GUI的内部API来执行实际操作
3. 把结果封装成JSON-RPC格式返回

### 3.2 JADX MCP Server的配置

这个Python程序需要这样启动：

```bash
python jadx_mcp_server.py --http --port 8651
```

启动后，它会在8651端口监听HTTP请求。

**我踩过的坑**：
1. 这个端口号不是8650，是8651（8650是JADX插件自己的端口）
2. Python需要安装特定的依赖包
3. Windows和Linux的路径格式不一样

### 4.2 第一步：用户在前端操作

前端页面`jadx_mcp.html`是一个聊天界面，用户可以：
1. 配置JADX MCP服务器地址
2. 输入AI的API Key
3. 输入问题，比如"这个APK用了什么网络库？"
4. 点击发送

前端发送请求的代码大概是这样的：

```javascript
// web/jadx_mcp.html

// 发送消息
async function sendMessage() {
    const message = document.getElementById('input').value;
    
    // 建立SSE连接，实时接收AI的响应
    const eventSource = new EventSource(`/mcp/jadx/chat/stream?sessionId=${sessionId}`);
    
    // 发送POST请求，带上用户的消息
    await fetch('/mcp/jadx/chat/stream', {
        method: 'POST',
        headers: {'Content-Type': 'application/json'},
        body: JSON.stringify({
            sessionId: sessionId,
            message: message
        })
    });
```

### 4.3 第二步：Go后端接收请求

后端代码`jadx_stream.go`的`JADXChatStreamHandler`函数处理这个请求：

```go
func JADXChatStreamHandler(w http.ResponseWriter, r *http.Request) {
    // 1. 解析请求
    var req struct {
        SessionID string `json:"sessionId"`
        Message   string `json:"message"`
    }
    json.NewDecoder(r.Body).Decode(&req)
    
    // 2. 获取会话（会话里存了AI配置、MCP连接等信息）
    session := GetOrCreateJADXSession(req.SessionID)
    
    // 3. 设置SSE响应头（SSE就是让服务器能主动推送数据给浏览器）
    w.Header().Set("Content-Type", "text/event-stream")
    
    // 4. 在后台运行AI处理，同时向浏览器推送结果
    go func() {
        processChatStream(session, req.Message, controller)
    }()
    
    // 5. 持续推送AI返回的片段给浏览器
    for {
        select {
        case msg := <-controller.messageChan:
            fmt.Fprintf(w, "data: %s\n\n", msg)
            flusher.Flush()  // 立即推送
        }
    }
}
```

### 4.4 第三步：调用AI服务

`processChatStream`函数会构建消息，发送给AI：

```go
func processChatStream(session *JADXChatSession, userMessage string, controller *StreamController) error {
    // 1. 构建发送给AI的消息
    messages := []ChatMessage{
        {Role: "system", Content: "你是一个APK分析助手..."},
        {Role: "user", Content: userMessage},
    }
    
    // 2. 发送给AI（以DeepSeek为例）
    provider := NewDeepSeekProvider(apiKey)
    
    // 3. AI可能会返回普通文字，也可能会调用工具
    response, toolCalls, err := provider.Chat(messages, tools)
    
    // 4. 如果AI要调用工具（比如要反编译APK）
    if len(toolCalls) > 0 {
        // 调用JADX MCP获取结果
        mcpResult := callJADXTools(toolCalls)
        
        // 把工具返回结果再发给AI
        messages = append(messages, response, mcpResult)
        finalResponse, _, _ := provider.Chat(messages, nil)
        
        // 推送最终结果
        controller.Send(finalResponse)
    } else {
        // 直接推送AI的回复
        controller.Send(response)
    }
}
```

```go
func callJADXTools(toolCalls []AIToolCall) ChatMessage {
    results := []
    
    for _, tc := range toolCalls {
        switch tc.Name {
        case "get_class_list":
            // 调用JADX MCP获取类列表
            result, _ := mcpConn.SendMCPRequest("tools/call", map[string]interface{}{
                "name": "get_class_list",
                "arguments": tc.Arguments,
            })
            results = append(results, formatResult(result))
            
        case "decompile_class":
            // 反编译指定类
            result, _ := mcpConn.SendMCPRequest("tools/call", map[string]interface{}{
                "name": "decompile_class", 
                "arguments": tc.Arguments,
            })
            results = append(results, formatResult(result))
        }
    }
    
    // 返回格式化的结果给AI
    return ChatMessage{
        Role: "tool",
        Content: strings.Join(results, "\n---\n"),
    }
}
```

## 六、关键代码说明

### 6.1 MCP连接代码（jadx.go）

```go
// JADXMCPConnection 是JADX MCP连接的封装
type JADXMCPConnection struct {
    mu       sync.RWMutex
    baseURL  string  // MCP服务器地址，如 http://127.0.0.1:8651
    client   *http.Client
    connected bool
    tools    []MCPTool  // 可用工具列表
    sessionID string    // Session ID（有些MCP服务器需要）
}

// SendMCPRequest 发送JSON-RPC请求
func (conn *JADXMCPConnection) SendMCPRequest(method string, params map[string]interface{}) (map[string]interface{}, error) {
    // 1. 构建请求
    requestID := conn.lastRequestID + 1
    request := map[string]interface{}{
        "jsonrpc": "2.0",
        "id":      requestID,
        "method":  method,
        "params":  params,
    }
    
    // 2. 序列化为JSON
    reqData, _ := json.Marshal(request)
    
    // 3. 发送HTTP POST
    req, _ := http.NewRequest("POST", conn.baseURL+"/mcp", bytes.NewReader(reqData))
    req.Header.Set("Content-Type", "application/json")
    req.Header.Set("Accept", "application/json, text/event-stream")
    
    // 4. 处理响应
    resp, err := conn.client.Do(req)
    // ...
}
```

### 6.2 SSE推送代码（jadx_stream.go）

```go
// JADXChatStreamHandler 处理聊天请求，同时通过SSE推送结果
func JADXChatStreamHandler(w http.ResponseWriter, r *http.Request) {
    // 设置SSE响应头
    w.Header().Set("Content-Type", "text/event-stream")
    w.Header().Set("Cache-Control", "no-cache")
    w.Header().Set("Connection", "keep-alive")
    
    flusher := w.(http.Flusher)
    
    // 启动后台处理
    go processChatStream(session, message, controller)
    
    // 持续推送结果
    for {
        select {
        case msg := <-controller.messageChan:
            // SSE格式：data: 开头，两个换行结尾
            fmt.Fprintf(w, "data: %s\n\n", msg)
            flusher.Flush()  // 关键！立即推送给浏览器
        case <-controller.abortChan:
            return
        }
    }
}
```

## 七、常见问题解答

### Q1: 为什么不用现成的库？

A: MCP协议相对较新，Go语言的支持库不多。而且我们的需求比较简单（只是HTTP+JSON），自己实现更灵活。

### Q2: 为什么需要自动启动服务器？

A: JADX MCP Server是一个独立的Python程序，需要用户手动启动。但我们的系统设计是"用户打开网页就能用"，所以加了自动检测和启动的功能。

### Q3: SSE和WebSocket有什么区别？

A: SSE只能服务器推送给浏览器，WebSocket是双向的。但我们的场景只需要单向推送，所以用更简单的SSE。

### Q4: 工具调用是怎么触发的？

A: 这是AI的决策。当AI分析一段代码后，如果觉得需要更多信息（比如查看某个类的源码），它会返回一个"工具调用"的响应。后端代码会识别这个响应，自动调用相应的MCP工具，然后把结果发回给AI继续分析。

## 八、总结

这个项目的主要工作量在于：

1. **理解MCP协议**：这不是看书就能懂的，需要看官方文档、查资料、尝试理解这个协议的设计思想

2. **串接多个服务**：我们的系统涉及浏览器、Go后端、MCP服务器、AI服务四个组件，每个组件的通信方式都不一样，需要理解它们是怎么交互的

3. **处理边界情况**：网络超时怎么办？服务器没启动怎么办？AI返回格式不对怎么办？这些都需要处理

4. **实时推送**：用SSE实现实时推送，比普通HTTP请求复杂很多

代码量确实不多，但背后的技术原理和系统设计是花了很多时间研究的。
