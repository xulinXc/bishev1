## 示例1：ThinkPHP RCE漏洞

### 请求

```bash

  -H "Content-Type: application/json" \
  -d '{
    "provider": "deepseek",
    "apiKey": "your-api-key-here",
    "model": "deepseek-chat",
    "autoVerify": true,
    "targetURL": "http://vulnerable-target.com",
    "maxRetries": 5,
    "exp": {
      "name": "ThinkPHP 5.0.23 RCE",
      "exploitSuggestion": "ThinkPHP 5.0.23 远程代码执行漏洞，通过invokefunction调用system函数",
```

## 常见问题

### Q1: 验证一直失败怎么办？

A: 检查以下几点：
1. 目标是否真的存在漏洞
2. 网络是否可达
3. 查看verifyLogs中的详细错误信息
4. 手动运行生成的EXP进行调试

### Q2: 如何调整重试次数？

A: 在请求中设置 `maxRetries` 参数：
```json
{
  "maxRetries": 3,
  ...
}
```

### Q3: 生成的EXP保存在哪里？

A: 默认保存在 `/tmp/neonscan_exp/` 目录，文件名格式为 `exp_attempt_N.py`

### Q4: 如何查看详细的验证日志？

A: 响应中的 `verifyLogs` 字段包含所有验证日志，也可以在服务器控制台查看实时输出

### Q5: 支持哪些AI提供商？

A: 目前支持：
- DeepSeek (推荐)
- OpenAI
- Anthropic (Claude)
- Ollama (本地)
